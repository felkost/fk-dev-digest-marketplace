# Usage-stats backend (`server/`)

A small Fastify + Drizzle + PostgreSQL service that counts how the catalog website is used —
copy-install clicks and plugin-page views — and serves the aggregates back. Deliberately built on
the same stack this marketplace's own `engineering-paved-path` plugin teaches (Fastify, Drizzle,
Zod, PostgreSQL): the marketplace dogfoods its own paved path.

Privacy by construction: an event is only `{ type, plugin }` — no IP, no user agent, no cookies,
no session ids are ever stored.

## API

| Route | Method | Description |
|---|---|---|
| `/api/events` | POST | Record one event. Body: `{ "type": "copy_install" \| "plugin_view", "plugin": "<kebab-case-name>" }`. Strict schema (unknown fields rejected), rate-limited 30/min per IP. Returns 204. |
| `/api/stats` | GET | Aggregated counts: `{ generatedAt, perPlugin: { <plugin>: { copy_install, plugin_view } }, totals }`. Cached 60 s. |
| `/healthz` | GET | Liveness + database check. |

CORS allows only the origins in `CORS_ORIGINS`; everything else is browser-blocked.

## Environment

| Variable | Required | Meaning |
|---|---|---|
| `DATABASE_URL` | yes | PostgreSQL connection string (Neon free tier works; SSL is honored from the URL). |
| `CORS_ORIGINS` | no | Comma-separated allowed origins. Defaults to the GitHub Pages and Render site origins. |
| `PORT` | no | Listen port (Render sets it; defaults to 3001 locally). |

## Local development

```
cd server
npm ci
npm run db:generate        # only after changing src/db/schema.ts
DATABASE_URL=postgres://... npm run dev
```

Migrations live in `drizzle/` (generated by drizzle-kit, committed) and are applied automatically
on boot — no separate migration step in deployment.

## Deployment (Render + Neon)

Wired in the repo-root [`render.yaml`](../render.yaml) as the `fk-dev-digest-stats` web service
(`rootDir: server`, build `npm ci && npm run build`, start `npm run start`). One-time setup:

1. Create a free PostgreSQL database at [neon.tech](https://neon.tech) and copy its connection
   string.
2. In the Render dashboard, open the `fk-dev-digest-stats` service → Environment → set
   `DATABASE_URL` to that string (`sync: false` in the blueprint keeps it out of git).

The website picks the backend up through `VITE_STATS_API` at build time (set in `render.yaml` for
Render and in `.github/workflows/pages.yml` for GitHub Pages). If `VITE_STATS_API` is unset, the
site builds with stats collection fully disabled and works exactly as before.

## What this does not do (yet)

GitHub clone counts — the closest proxy for real `/plugin install` runs (installs clone the repo,
they never touch the website) — are not harvested yet. The GitHub Traffic API keeps only 14 days
of history, so a scheduled job (GitHub Action cron writing into this database or into a committed
JSON) is the natural next step.
