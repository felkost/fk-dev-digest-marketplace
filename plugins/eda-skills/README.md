# EDA-скіли: як застосовувати

Система з **4 взаємопов'язаних скілів**, що перетворює сирі дані на задокументований,
leakage-safe, валідований датасет для деревної / нейронної / генеративної моделі.
Наголос — на EDA і **якості датасета**; тренування та тюнінг фінальної моделі —
поза межами (прості моделі/AE·VAE/pretrained encoders дозволені лише як
diagnostic probes для вивчення даних).

```
plan-eda-dataset            ← оркестратор: план, split-стратегія, візуалізація, readiness gate
├── audit-eda-data-quality      ← якість, розподіли, дисбаланс (діагноз), leakage
├── discover-eda-structure      ← кореляції/MI, кластери, зниження розмірності, час
└── engineer-select-eda-features← ознаки, mRMR/wrapper, балансування (in-fold), валідація
```

Кожен скіл: `SKILL.md` (методологія, укр.) + `agents/openai.yaml` (пакет для
Codex/ChatGPT) + `references/*.md` (поглиблені довідки) + `scripts/*.py`
(leakage-safe утиліти). Оркестратор має ще `assets/eda_workflow_template.ipynb`,
`scripts/contracts.py` (спільна схема маніфестів), `scripts/eda_plots.py` +
`references/visualization.md` (візуалізація етапів і результатів EDA:
розподіли, missingness, кластерні діагностики, importance, порівняння проб)
і `scripts/insights.py` (persistent-правила користувача в `insights.md`).

## Спосіб 6 — уточнення користувача, що переживають сесію (`insights.md`)

Доменні правила, які скіл **не може вивести з даних** («пропуск тиску = датчик
вимкнено, не імпутувати», «тут DBSCAN, а не KMeans»), зберігаються в
`insights.md` і застосовуються автоматично в наступних сесіях — замість того щоб
користувач повторював те саме виправлення щоразу.

```python
from insights import append_insight, applicable, format_for_prompt

# записати виправлення користувача (why — обов'язково: без причини правило не перенести)
append_insight(rule="Use DBSCAN, not KMeans",
               scope="discover-eda-structure/clustering",
               why="Машини простоюють одним щільним згустком; KMeans ріже його штучно")

# прочитати перед етапом (stage-wide + method-specific правила)
print(format_for_prompt(applicable("discover-eda-structure/clustering")))
```

Як це працює:

- **Формат** — markdown, редагується вручну; скіл не переписує його поверх ваших правок.
- **Scope** — `<скіл>/<метод>`; правило діє лише у своїй області, не глобально.
- **Переказ перед дією** — скіл озвучує застосовне правило («застосовую ваше правило від …»), щоб ви впіймали застаріле чи хибно застосоване.
- **Пріоритет** — hard-інваріанти вищі за insight: правило на кшталт «фітити scaler на всіх даних включно з test» записується зі `status: conflicts_invariant`, **не застосовується** й виноситься на обговорення з безпечною альтернативою.
- **Перегляд** — `supersede(id)` замість видалення: історія міркувань пояснює, чому нове правило саме таке.

## Розпаралелювання субагентами (коли це доречно)

Етапи 2–4 **послідовні за побудовою**: аудит може змінити одиницю спостереження,
target або split — і це перевизначає вхід наступних етапів. Їх паралелити не можна.

Паралелиться **fan-out незалежних read-only питань усередині одного етапу**
(макс. 3–4 субагенти), коли питань ≥ 3 і кожне дороге:

| Етап | Гілки для fan-out |
|---|---|
| audit | неперетинні групи колонок (схема/пропуски/розподіли/викиди); окремо — цілісність зображень |
| discover | асоціації/redundancy \| кластери \| зниження розмірності \| часові залежності |
| engineer | незалежні сімейства ознак; або filter \| wrapper \| embedded як конкурентні докази |
| мультимодальні | по гілці на модальність |

Правила: split **спершу й заморожений** (усім гілкам той самий dataset contract +
split manifest + insights; субагент не проєктує свій split і не торкається test);
гілки повертають **маніфести/findings, а не змінені дані**; усе, що фітить спільний
трансформер, лишається в основному агенті (дві гілки з власними scaler = розбіжні
конвеєри); обов'язковий **merge-крок** — звести, зняти дублікати, розв'язати
суперечності. На малих датасетах fan-out коштує дорожче за виграш — за
замовчуванням послідовно.

## Робочий процес (як 4 скіли ланцюжок)

1. **plan** — контракт датасета, immutable raw, вибір split, план і readiness-критерії.
2. **audit** — схема, цілісність, пропуски, розподіли, викиди, **діагноз** дисбалансу, leakage, візуалізація знахідок. → передає blockers/warnings і підтверджений split.
3. **discover** — асоціації/MI, кластерована кореляція, кластери (вибір k за кількома сигналами), зниження розмірності, часові залежності. → передає каталог кандидатних ознак/репрезентацій.
4. **engineer** — побудова ознак, filter/mRMR/wrapper-відбір, **балансування in-fold**, diagnostic probes, візуалізація, фінальна валідація. → видає маніфести + dataset card + вердикт.
5. **plan → readiness gate** — `ready` / `ready_with_accepted_limitations` / `not_ready`. Якщо `not_ready` — повернутися лише на етап, що усуває названий дефект.

Етапи обмінюються 8 маніфестами (dataset contract, split, feature, transformation,
selection, sampling, diagnostic, dataset card) — див. `plan-eda-dataset/scripts/contracts.py`.
Фінальний звіт містить **візуальний додаток** зі збережених графіків етапів.

## Межі (важливо)

- **Дозволено як probe:** проміжні моделі, importance/SHAP, MI, кластери, ресемплінг — **для вивчення даних** (fit на train, сигнал з OOF/validation).
- **Поза межами:** тренування/тюнінг фінальної моделі; споживання **запечатаного test** під час EDA; трактування importance/SHAP/кореляції/MI/кластерів як **причинності**; fit на validation/test; **примусовий 50/50** за замовчуванням.
- **Результат EDA** = валідований датасет + маніфести + діагностичні докази, **не** фінальна модель і не test-метрика.

## Ключові діагностики (пастки, які скіли ловлять)

- **Ціле диференціювання купує стаціонарність ціною пам'яті.** Заміряно на випадковому блуканні (n=5000): при `d=1` кореляція з початковим рівнем — **0.013** (рівень знищено), а при `d=0.30` ADF уже проходить (p=0.017) і кореляція лишається **0.888**. `frac_diff` причинний (зміна `x[500]` не рухає жодного виходу раніше 500), ваги залежать лише від `d` — але **вибір `d` робиться лише на train**. Пропуски фатальні й тихі: gap кожні 10 рядків при ширині вікна 11 дав **нуль** скінченних виходів, тому `min_frac_diff_order` віддає `gaps_block_positive_d`. Обране `d` — **не оцінка порядку пам'яті**: на ARFIMA з відомим d₀=0.50/0.60/0.70 сканування повертало 0.00, бо ADF майже не має потужності проти дробових альтернатив.
- **«25-й перцентиль» — не одне число.** Інтерполяційних конвенцій дев'ять, і інструменти розходяться **за замовчуванням**: `numpy.percentile`/`pandas.quantile` дають те саме, що stdlib-`statistics.quantiles(method="inclusive")` (перевірено до збігу float при n=9/11/25/101), тоді як власний дефолт `statistics.quantiles` — `"exclusive"`, тобто інше число. Розрив між цими дефолтами: **21.3% sd при n=8, 31.1% при n=12, 14.1% при n=20**, далі 1.0% при n=50 і 0.01% при n=10000. Дев'ять методів numpy дають розкид q25 у 23.4% sd при n=10. **Значення сходиться, а рішення — ні**: межі IQR розходяться хоча б на одній точці у 21.5%/15.4%/13.1%/12.4%/12.8% випадків при n=10/50/200/1000/5000 (нормальні) і ~25% на логнормальних — виходить на плато, а не зникає, бо зі зростанням n більше точок опиняється у вужчому проміжку. `quantile_convention_report` дає три стани, і середній (`borderline_points_only`) — не дефект, а привід не вішати незворотну дію на одну прикордонну точку.
- **ROC-AUC не «непридатний для дисбалансу» — він відповідає на інше питання.** Це ймовірність, що випадковий позитив стане вище випадкового негатива: коректно за будь-якої поширеності. Саме інваріантність робить його придатним для порівняння моделей між популяціями з різними базовими частотами — і непридатним для оцінки черги алертів. Дзеркальна пастка поради «беріть PR-AUC»: **AP не порівнюваний між популяціями** — той самий скорер дає AP 0.869→0.041 лише через зміну поширеності. TPR і FPR нормуються кожен у своєму класі, тож проріджування негативів не змінює AUC (виміряно 0.873/0.874/0.866/0.871/0.868 при поширеності 0.5/0.2/0.05/0.01/0.002), а average precision на **тих самих** скорах падає 0.869→0.041. При фіксованому **AUC ≈ 0.95** precision@recall=0.8 становить 0.919/0.546/0.111/0.012 при поширеності 0.5/0.1/0.01/0.001 — тобто **від 1.1 до 89 алертів на один справжній випадок**. Ємність черги вирішувати за `alerts_per_true_positive`, не за AUC.
- **Перемішані фолди на мітці з forward-вікном вигадують уміння з нічого.** Виміряно на дизайні, де істинна передбачуваність рівно нульова (мітка = сума наступних `h` інновацій, ознаки — лише з минулого): shuffled 5-fold дає кореляцію **+0.151/+0.203/+0.205** при h=5/20/50, суміжні фолди — −0.026/−0.048/−0.116. Шкода перемішування — не межа фолду, а train-рядки *між* test-рядками; тому суміжність робить майже всю роботу, а `purge_and_embargo` знімає крайовий залишок. Contiguous k-fold при великому `h` має ще й **від'ємне** зміщення, тож одне число CV ненадійне в обидва боки.
- **Uniqueness і design effect для мітки на вікні — різні числа.** `1/h` — правильна **вага** спостереження; інфляція дисперсії — `1+(h−1)(2h−1)/(3h)` (→`2h/3`). Підстановка `h` замість формули занижує alpha до 0.011 при номінальних 0.05 і тихо викидає справжні ознаки; без жодної корекції при h=50 **74%** незалежних пар колонок читаються як значущі. При h=1 формула дає рівно 1.0, тоді як межа `2h/3` брехала б 0.67 на даних без перекриття взагалі.
- **Мітка від «90%-точного» детектора при рідкісному класі — переважно хибні позитиви.** При sens=spec=0.90 і поширеності 0.02: PPV = **0.155** (85% позитивних міток хибні), а частка позитивних у даних читається як 0.116 замість 0.02 — **роздування 5.8×**, яке успадковує будь-яке рішення про дисбаланс. Важіль при рідкісному класі — **специфічність, не чутливість**: 0.80/0.999 дає PPV 0.890. Невизначеність поширеності домінує над невизначеністю заявленої точності на порядок (0.318→0.336 проти 0.692→0.155).
- **Вузол-диз'юнктний split ребер коштує пів графа, і про це ніхто не попереджає.** Якщо відкласти частку `q` вузлів, ребра діляться як `(1−q)²` train, `q²` test, `2q(1−q)` — крізь межу й непридатні. Заміряно проти передбачення на графі з 1200 вузлів: при `q=0.2` тест отримує **4.0%** ребер, а **31.8%** викидається; щоб дійти до 20% ребер у тесті, треба відкласти ~45% вузлів і втратити **49.5%** ребер. Бюджет на це закладати до того, як пообіцяти розмір тестової вибірки (`graph_split`).
- **Випадковий split ребер не є «новими даними».** Він відповідає на інше питання — «нове ребро між уже відомими вузлами», і це легітимно, якщо так само працює продакшн. Заміряно на 20 випадкових графах по 1200 вузлів: обидва кінці тестового ребра вже присутні в train у **98.2% / 100.0% / 100.0%** випадків при щільності 0.005 / 0.01 / 0.05 (найгірший граф — 96.6%). Помилка не в тому, що split випадковий, а в тому, що його звітують як узагальнення на нові сутності.
- **9505 ребер несуть точність ~533 спостережень.** Діади не є незалежними: кожне ребро при вузлі `i` тягне за собою те, чим є `i`. Заміряно на дизайні з відомою залежністю (значення діади = `z_i + z_j + шум`): номінальний 95% CI, порахований по таблиці ребер, покриває істину у **35.0–55.0%** випадків, а `n_eff` виходить **≈0.6 × кількість вузлів**, не кількість ребер. Node-jackknife (`dyadic_design_effect`) відтворює істинний deff у межах ~15% і помиляється **вгору**, тобто консервативно. Збирати більше ребер серед тих самих акторів дає значно менше, ніж здається.
- **Ненапрямлений граф, збережений двічі, виглядає напрямленим.** `(a,b)` і `(b,a)` в одному файлі дають рівно подвійну кількість ребер, подвоєний degree і вдвічі занижену щільність. Заміряно: 1752 сирих ребра на 876 справжніх. Сигнатура — reciprocity рівно `1.000`; `profile_graph` її ловить і навмисне **не** звітує це ще й як «дублікати ребер», щоб ніхто не «почистив» коректний edge-list. Окремо: **ізольовані вузли в edge-list невидимі за побудовою** — кількість вузлів звідти є нижньою межею, а не фактом.
- **1500 рядків можуть містити 183 незалежних спостереження.** Якщо рядки вкладені (користувачі, пристрої, школи, сесії), `n` перестає бути кількістю інформації. Заміряно при 60 кластерах × 25 рядків і ICC = 0.30: номінальний 95% CI покриває істину у **48.8%** випадків. `sampling_design.py` дає `n_eff = n/deff`; це успадковують усі CI, bootstrap, p-value і PSI-гард на ≥50 рядків у бакеті. Обмеження паніки: кластеризація б'є по контрастах **між** кластерами і не чіпає контрасти всередині (0.049 проти 0.359 при ICC 0.30).
- **Дві незалежні колонки «корелюють», якщо обидві варіюють на рівні кластера.** Заміряно на незалежно згенерованих x та y зі спільною лише структурою кластерів: 0.259 хибно значущих при ICC 0.30 і **0.537** при ICC 0.60. Корекція — `effective_n_for_association`, deff = `1 + (m0−1)·ICC_x·ICC_y`, тобто **добуток** ICC; підстановка сюди deff середнього перекориговує до 0.002 і тихо викидає справжні ознаки. Group-aware split цього не лікує — він проти leakage, а не проти інференсу.
- **Оцінка ознаки зміщена вгору саме тому, що ознаку відібрали.** При n=200 найкраща з 1000 суто шумових колонок має |r| = 0.241 і наївний p = 6e-4, причому p «переможця» задає **кількість кандидатів**, майже не n. При 5 справжніх ознаках серед 1000 кандидатів у топ-5 потрапляє в середньому 1.50 справжніх, а топ-1 є чистим шумом у 42.7% випадків. `expected_max_noise_correlation` дає шумову межу, `selection_inflation_probe` — чесну переоцінку; `inflation` ≈ 1 у справжніх ознак і ≈ 3.8× у відібраного шуму.
- **Мультиверс аналітичних рішень коштує 30% хибних відкриттів, і більше даних не рятує.** 18 захищуваних варіантів (3 правила викидів × 3 трансформації × 2 підгрупи) на **незалежних за побудовою** даних: 0.292/0.300/0.296 при n = 100/500/2000 проти 0.05 для одного фіксованого шляху. Рівень не падає з ростом n, бо це відбір, а не дисперсія. `1−(1−α)^k` тут — верхня межа, не оцінка.
- **Не обирати тест, попередньо протестувавши його передумову.** Двоетапна процедура (Левен → pooled/Welch) дає 0.045–0.067, але вирішальне інше: pooled t за нерівних дисперсій і нерівних груп хибний в обидва боки — 0.325 і 0.001, залежно від того, у більшої групи менша чи більша дисперсія. Welch номінальний скрізь, тож попередній тест не купує нічого.
- **Target — теж проксі, і найдорожчий.** `is_fraud` зазвичай означає *виявлене й підтверджене* шахрайство, `churn` — чиєсь правило 90 днів, `arrest` ≠ `crime`. Два різні дефекти: недопокриття (мітка бачить лише частину конструкта) і контамінація (мітка домішує щось інше й розділити вже не можна). Фіксувати конструкт, операційне означення і розрив між ними в маніфесті датасету.
- **Найважливіша ознака без механізму** (sampling weight на кшталт census `fnlwgt`, ID, індекс рядка) → metadata leakage або impurity bias дерев. Перевірити моделлю з іншим зміщенням (L1-лінійна після power transform проти дерев): `noise_probe_importance`, `null_target_importance`.
- **Шумові ознаки в топі importance, та ще й CV score зріс** → overfit протоколу, а не сигнал. Канарки (`noise_probe_importance`) роблять це видимим.
- **Дисбаланс може бути артефактом дублікатів** → `class_duplicate_report` рахує duplicate rate по класах і співвідношення до/після дедуплікації.
- **Хвостовий індекс Хілла бреше на логнормальних даних.** `Lognormal(0,2)` дає `α ≈ 1.05`, що наївно читається як «нескінченне середнє» — хоча всі моменти існують. Розділяє їх лог-шкала: логарифм Pareto це експоненційний розподіл (асиметрія ≈ +2, надлишковий ексцес рівно 6), логарифм логнормального — нормальний (≈ 0), логарифм log-Cauchy моментів не має (ексцес у сотнях). `family_router.tail_index` повертає `tail_family`, і діяти треба саме за ним, а не за `α`.
- **Маршрутизатор, що завжди має відповідь, є генератором випадкових назв.** `family_router.route` мовчить на добропорядних даних і на сімействах, які вже є на чарті Leemis; кандидата видає лише з тригером-виміром і посиланням, а перед прийняттям вимагає зіставити параметризацію, перевірити на відкладених даних проти простішого сімейства і назвати механізм.
- **Голий `log` на даних із нулями виробляє `-inf`, який не рахується як пропуск.** Між `b=1` і `b=2` лежить compound Poisson-gamma (Tweedie): маса в нулі + неперервна додатна частина — витрати на клієнта, страхові виплати, дохід на користувача. Заміряно: `b=2.02` при 30.4% нулів, тобто форма каже «log», а результат — `-inf` у 3804 з 12500 рядків. `mean_variance_scaling` звітує `zero_fraction` і в такому разі веде до `log1p` / hurdle-моделі / Tweedie GLM. «Чи була транзакція» і «скільки за умови, що була» — два різні питання.
- **Log-трансформація за скошеністю лікує симптом.** `mean_variance_scaling` оцінює `b` у `Var ≈ mean^b` по реальних групах: `b≈0` — адитивний шум (нічого не робити), `b≈1` — лічильні (sqrt), `b≈2` — мультиплікативний процес (**log**). Заміряно на симуляціях: 0.01 / 1.03 / 2.07. Логарифм виправданий механізмом — логнормальний замкнений щодо множення так само, як нормальний щодо додавання.
- **Overdispersion — це зазвичай пропущена коваріата, а не хибне сімейство.** `count_dispersion` рахує VMR: пуассонівська інтенсивність, яка сама варіює, і є gamma-Poisson сумішшю (маргінал — негативний біноміальний). Спершу шукати змінну, що пояснює варіацію. `VMR<1` — обмежений процес: біноміальний має `VMR = 1−p` конструктивно (перевірено: 0.51 для `Binomial(10, 0.5)`). Надлишок нулів проти `exp(−mean)` — питання **структурних нулів**, тобто визначення даних.
- **Сумувати можна завжди, лишитись у тому ж сімействі — ні.** Згортка на чарті Leemis позначена умовно: `C_p` (біноміальний — лише за фіксованого p), `C_α` (гамма — лише за спільного scale). За дрейфу ставки тижневий підсумок виглядає нормально, а CI й контрольні межі на ньому хибні. `rate_homogeneity` перевіряє це перед агрегацією; тест відкалібрований, тому ~1 із 20 однорідних рядів позначається помилково — читати `rate_min`/`rate_max` поруч із p-value.
- **Чарт замикань не знає про існування моментів.** Коші має властивість C (суми Коші лишаються Коші), нормальний має L — обидва замкнені щодо додавання, але границею ЦГТ є лише нормальний. Заміряно: std середнього для Коші не спадає від n=10 до n=100 000, для нормального спадає рівно як 1/√n. На важких хвостах `mean ± z·se` мовчки бреше — використовувати `bootstrap_ci` / `permutation_test_groups`.
- **`inf` — це не пропуск.** `pd.isna()` його не рахує, тож колонка проходить як «0% пропусків», а mean/std/квантилі мовчки стають NaN. `profile_schema` звітує `n_inf` окремо від `n_missing`; типове джерело — ділення на нуль у похідній ознаці.
- **Значення-заглушка виглядає валідним** (`email@company.com`, `1900-01-01`, `0` від upstream-системи) → сигналу пропуску немає взагалі. Ловиться концентрацією: `constant_value` / `dominant_value_share`. Дата-сентинел усередині правдоподібного діапазону перевірку меж ніколи не провалить.
- **Одруківку й легітимну відмінність не розділяє поріг схожості**: `apples`/`appels` = 0.833, але `xl`/`xxl` = 0.800. Розділяє довжина рядка — тому `near_duplicate_categories` порівнює fuzzy лише мітки від 4 символів, а результат — черга на перегляд, не автозаміна.
- **Невалідне значення створює наша ж трансформація** → `time_in_app = finish_hour − start_hour` дає −22 для сесії 23:00→01:00. Видалення таких рядків зміщує вибірку в бік сесій, що не перетинають північ: лагодити треба формулу. `logical_constraints` запускати після кожного кроку feature engineering, не лише на сирих даних.
- **Один ключ — два товари** (`monitor stand` за $999 під тим самим product ID) → `key_attribute_conflicts`. Розрізняти помилку вводу, slowly changing dimension (справжній ключ `(id, valid_from)`; згортання в один рядок вносить майбутнє в минулі рядки) і перевикористаний ID.
- **Дублікати не є дефектом автоматично** → повторні вимірювання однієї сутності лікуються group-aware split, а не видаленням. `duplicate_report` показує, які колонки конфліктують: без конфліктів можна залишити один рядок, інакше потрібне правило «найсвіжіший» або злиття.
- **Дефект, що відтворюється при наступному імпорті** (дубль-вставка на ретраї, mojibake від неправильного кодека, join, що розмножує рядки) → чистити датасет марно, це задача на пайплайн. Походження дефекту фіксувати в звіті поруч із severity.
- **Одна крива не обирає k** → `k_scan` (SSE-elbow із програмним knee + silhouette + DB/CH + BIC/AIC) плюс `cluster_stability` і `silhouette_profile`. Кластерів більше, ніж класів — валідний результат (внутрішньокласові режими).
- **Занадто хороша одиночна ознака** → target або post-outcome proxy ще в матриці (`leakage_checks`).
- **Нормальність залишків Гаусс–Марков не вимагає.** Заміряно покриття 95% CI на 6000 повторах: за збалансованого дизайну логнормальні похибки навіть при n=10 дають 0.947. Ламається рівно одна конфігурація — мале n **і** важкохвостий `x` **і** ненормальні похибки разом (0.925). Тому діагностувати треба **важелі** (`leverage_diagnostics`, прапорець `clt_safe`), а не нормальність залишків: коли кілька рядків тримають дизайн, усереднення не відбувається і ЦГТ не вмикається.
- **«Провести лінію через хмару» має три різні відповіді.** При r=0.30 нахили: `y~x` = 0.298, `1/(x~y)` = 3.246, PC1 = 0.946. OLS мінімізує залишки вертикально, око тягнеться до перпендикулярної підгонки — тобто до PC1. Регресійний нахил завжди пологіший за візуальний (це і є регресія до середнього), а компонента PCA не є регресійною прямою.
- **Обумовлення на колонку змінює питання, а не «очищає» дані** — і жодна кореляція їх не розрізняє (заміряно на 40 000 рядків). Конфаундер (`Z→X, Z→Y`) і медіатор (`X→Z→Y`) виглядають майже однаково в даних (corr(Z,X)≈corr(Z,Y)≈0.57), але обумовлення на Z дає правильний нуль у першому й знищує реальний ефект 0.49 у другому. Колайдер (`X→Z←Y`) — гостріший: X і Y незалежні (маргінальна кореляція 0.006), проте обумовлення на їхньому спільному наслідку Z **вигадує** зв'язок −0.50 там, де істина 0 (парадокс Берксона). Перед «прибрати вплив колонки» — з'ясувати, чи вона причина предиктора, наслідок, чи спільний наслідок предиктора й цілі.
- **Скоригована різниця груп (ANCOVA) припускає однаковий нахил коваріати в кожній групі.** Заміряно на дизайні з перетином нахилів (0.3 проти 1.0, перетин при Z≈2.86): справжня різниця груп −1.30 при Z=1 і +4.30 при Z=9, а модель зі спільним нахилом видає одне число +1.51, що не збігається з істиною в жодній точці й ховає зміну знаку. Перевіряти взаємодію група×коваріата, перш ніж звітувати одну скориговану різницю.
- **Колінеарність серед контрольних змінних нікому не шкодить.** Заміряно на 400 повторах: sd фокусного коефіцієнта 0.0595 при незалежних контролях і 0.0586 при контролях з кореляцією 0.99 між собою. Глобальний `vif_prune` у такому наборі видаляє дві нешкідливі змінні задарма — використовувати `vif_prune(..., focal=[...])`, який ріже лише те, що дістає до фокусних предикторів.
- **Нестабільність коефіцієнта — не зміщення, і прогноз від неї не страждає.** Заміряно: зростання `corr(x1,x2)` з 0.0 до 0.999 роздуло sd коефіцієнта у 23 рази (0.050 → 1.138), а RMSE на відкладених даних не змінився (1.003 → 1.002) — і це лінійна модель. Широкий інтервал під сильним перетином є *правильною* відповіддю, а не дефектом: якщо предиктори несуть те саме, їхні унікальні внески справді нерозділювані.
- **Центрування полінома не лагодить нічого.** На `y ~ x + x²` VIF падає з 86.7 до 1.002, але R², усі прогнози, коефіцієнт при `x²` і його стандартна похибка побітово тотожні. Центрування прибирає артефакт вимірювання перетину навколо нуля — «покращений» VIF ніколи не свідчив про реальну проблему.
- **Чуже «VIF = 500» не інтерпретовне.** Нецентрована версія дає VIF ≈ 660 000 для трьох *незалежних* ознак із середнім 1000 і sd 1. Різниця в 600 000 разів породжена лише зміщенням змінних від нуля.
- **Мультиколінеарність невидима для heatmap.** Якщо `x ≈ w + z`, усі попарні `|r| ≈ 0.7` (нижче звичного порогу 0.8), а VIF — сотні. Тому `variance_inflation_factors` запускати **завжди**, а не лише «якщо кореляція висока»; прунити ітеративно (`vif_prune`) — видалення однієї ознаки зазвичай повертає весь блок під поріг.
- **Корельовані «близнюки» ділять importance** — жодна не виглядає важливою, хоча разом вирішальні. Перед видаленням «слабкої» ознаки перевірити, чи не в блоці вона, і тестувати блок груповою абляцією.
- **Сильна модель + чисті дані → ресемплінг майже не допомагає**; ваги/поріг зазвичай достатньо. Ресемплінг окупається для слабших моделей і зашумлених перетинних класів.
- **Малі групи на обох краях рейтингу часток** (найкращі/найгірші школи — малі школи) → `group_rate_funnel` + `rate_funnel`: заявляти відхилення лише поза біноміальним коридором.
- **Статистика без невизначеності — не доказ** → `bootstrap_ci`; дві групи порівнювати через `bootstrap_diff_ci` (перетин двох окремих 95% CI — це ~90%, не 95%).
- **p-value відповідає на питання про `n` не менше, ніж про дані — і хибить в обидва боки.** Великі `n`: критичне значення KS спадає як `1.36·√(2/N)`, тож на зсуві 0.005 sd (нічого не змінює) p-value = 0.70 / 0.65 / 0.39 / 0.19 / **0.0068** при N = 10²…10⁶ — дані ті самі, змінюється лише `n`; на продакшн-обсягах поріг за p-value стає сигналізацією, що не змовкає. Малі `n`: незначущість — не доказ відсутності ефекту, а «post-hoc power» тавтологічна (рангова кореляція з p дорівнює рівно **−1.000000**; максимум серед усіх незначущих досліджень — 0.5027). Міряти **величину**: `distribution_shift` (PSI, Wasserstein), `bootstrap_diff_ci`.
- **PSI має нульову межу, залежну від `n`** → пам'ятка «<0.1 стабільно / >0.25 сильний дрейф» без умови на `n` хибна: очікування PSI під нулем дорівнює `(B−1)·(1/n_ref + 1/n_cur)`, тож на **ідентичних** розподілах із 10 бінами PSI = 0.64 при n=50 і 0.20 при n=100. Читати `psi_ratio`, а сирий поріг застосовувати лише за `psi_reliable` (≥ ~50 рядків на бін).
- **Група, відібрана за екстремумом, «покращується» сама собою** → заміряно на даних, де між двома вимірами **не сталося нічого**: найгірші 10% «виросли» на **+7.08**, найкращі 10% «впали» на −6.91. `regression_to_mean` рахує очікуваний зсув і повертає `excess_change` (заміряно: за істинних ефектів 0/+3/+6/−2 повертає +0.03/+3.03/+6.03/−1.97). Опорну пряму будувати по **невідібраних** рядках: підгонка по всій вибірці з'їдає частину ефекту (з істинних +3.0 повертала лише +1.784).
- **Кореляція біля нуля — це питання, а не відповідь.** Шість різних механізмів дають `r ≈ 0` над реальним зв'язком (заміряно): перевернуте U (Pearson +0.002, але `corr(|x|,y)` = −0.907), зміна знаку в підгрупі (−0.012 проти +0.765/−0.766), реверс Сімпсона (об'єднано **+0.669**, у підгрупах ≈ −0.62 — хибний уже знак), огрублення цілі (0.597 → 0.238), обмеження діапазону (0.500 → 0.211), супресія. Near-zero клітина — привід побудувати графік пари, а не викинути ознаку.
- **Відфільтрована вибірка занижує кореляції конструктивно** → `range_restriction` (Торндайк Case II): на істині 0.50 верхні 5% дають 0.225, зворотна формула повертає 0.528. «SAT не передбачає GPA» серед зарахованих — арифметика, а не знахідка. Потрібен `sd` **ззовні** фільтра, інакше чесний висновок — «це нижня межа».
- **Множинність кусається швидше, ніж здається** → `1 − (1 − α)^k`; заміряно: 10 незалежних тестів = **40.6%** шансу хоча б однієї хибно-позитивної, а матриця на 10 ознаках містить 45 пар. Стосується **різних** тестів, не повторних запусків; кожен оцінений параметр — теж тест.
- **«ANOVA, а потім попарні тести» — захист, що зникає рівно тоді, коли потрібен** → під **повною** нульовою критерій найменшої значущої різниці (LSD) тримає 0.048–0.051 при будь-якому числі груп, бо сам омнібус спрацьовує лише у 5% випадків і вся робота — на воротах. Під **частковою** нульовою (одна група реально різна, решта однакові) омнібус значущий у **100%** повторів, і LSD лягає точно на нескориговану частоту: **0.1224/0.2036/0.2843/0.4374** при g=4/5/6/8. Саме часткова нульова і є реальним випадком EDA. `pairwise_group_differences` не залежить від омнібусу: Welch беззастережно, CI різниці, Hedges' g, **Holm** за замовчуванням (0.038–0.049 в обох режимах, потужніший за Бонферроні задарма). **BH тут не заміна** — 0.0863→0.1752, бо це інша гарантія (FDR, а не familywise).
- **Ціль, обчислена з власних ознак** → `composite_target_probe`. Індексні цілі часто *визначені* як функція сусідніх колонок: Happiness Score = `Economy + Family + Health + Freedom + Trust + Generosity + Dystopia Residual` рівно. Однофакторний скрин сліпий до цього: заміряно на структурі WHR окремі кореляції +0.36…+0.45 (у реальних даних 0.72–0.78) — жодна не переходить поріг, а разом дають `R² = 1.000000000000` і відносний залишок **1.07e-15** проти **0.0992** у чесної моделі з `R² = 0.99`. **Запобіжник важливіший за статистику**: при `n ≤ p+1` МНК відтворює будь-яку ціль точно (на чистому шумі мінімальний залишок 0.0000), тож проба відмовляється нижче `n ≥ 2(p+1)`.
- **«Видалити за 1.5·IQR і перебудувати boxplot» — цикл без нерухомої точки** → межа означена відносно того, що лишилося, тож кожен прохід породжує нові «викиди». Заміряно на чистих даних: логнормальний — 12 проходів, видалено **14.40%** рядків, sd 2.1433→**0.6866** (зникло 68% розмаху, і це був власний правий хвіст змінної); експоненційний — 7.70%; навіть нормальний — 0.73% уже на першому проході. «На boxplot немає точок» — твердження про boxplot, а не про дані.
- **Kurtosis міряє хвости, а не «гостроту піка»** → область `|z| < 1` містить 68.26% рядків і дає лише **3.75%** статистики. Пік і ексцес зазвичай ростуть разом (Spearman +0.94), тому пам'ятка живуча — але порядок інвертується: щільність у моді Лапласа 0.7071 проти 0.4901 у t(5), а ексцес +3.000 проти +6.000 (точні значення). Плюс вибірковий ексцес занижений саме на важких хвостах (t(5): 5.337 замість 6.000).
- **`√R²` — не коефіцієнт кореляції** → корінь невід'ємний, тож знак із `R²` не відновлюється: заміряно `r = −0.904853`, а `√R² = **+0.904853**`. У множинній регресії ще гірше — `√R² = 0.8375` не відповідає жодній парі (попарні кореляції `+0.5919/−0.5084/+0.2539`). `r` — для пари, `R²` — для моделі.
- **«n > 30 вистачає для ЦГТ» — це про симетрію, а не про `n`** → перекос середнього спадає як `skew(x)/√n`, тож при n=50 покриття 95% t-інтервалу становить 0.951 (нормальний), 0.934 (експоненційний), **0.900** (логнормальний(0,1)), **0.695** (логнормальний(0,2) — і лише 0.874 навіть при n=2000). Промах **односторонній**: інтервал сідає *під* істину у 9.4% випадків проти 0.28% над нею (логнормальний(0,1), n=50), бо мала правоасиметрична вибірка не містить великих значень, що тримають середнє. **Bootstrap не рятує**: percentile на малих вибірках *гірший* за t навіть на нормальній популяції (0.890 проти 0.951 при n=10), а BCa відіграє ~1 п.п. Вихід — більше рядків, інша оцінювана величина (медіана/усічене середнє) або модель сімейства, а не інша формула інтервалу.
- **Зміна в часі — не обов'язково зміна у світі** → перша альтернативна гіпотеза для будь-якого `distribution_shift` — **зміна інструмента** (нова форма, версія застосунку, міграція схеми), а не популяції. Плюс history, maturation, testing-ефект і attrition. Лаг — це змістовна гіпотеза: хибний горизонт ховає реальний ефект не гірше за його відсутність.
- **Пропуск може бути дизайном, а не дефектом** → planned missing designs (multiform, accelerated longitudinal, two-method) дають MCAR **за побудовою**; діагностична ознака — блокова, надто акуратна структура пропусків. «Ми використали FIML» не є описом даних: MAR — це припущення, а не властивість методу.
- **Рядки, яких немає взагалі** (виживання, MNAR-логи) → таблиця «наші клієнти» містить лише тих, хто не пішов; великі дані лікують дисперсію, але не зміщення відбору. Реконструювати рамку **на момент рішення** (point-in-time snapshot), а не на момент вивантаження; питати, що мало статися, щоб рядок сюди потрапив.
- **«Бімодальність», яка зникає при іншій кількості бінів** → `binwidth_sensitivity` перед заявою про мультимодальність.
- **χ²/Cramér's V каже «зв'язок є», але не де** → `quetelet_table`: які клітини тягнуть зв'язок і в який бік (χ²/N = середній Кетле-lift); базову популяцію називати явно.
- **z-score приглушує кластероносні ознаки** (std роздутий саме в мультимодальних) → для кластеризації порівняти зі `scale="range"`; `anomalous_clusters`/`ik_means` дають детерміновані K+центри, а аномальні одиночки — кандидати в data errors.
- **Accuracy 98% при prevalence 98% — нульовий lift** → звітувати `precision/prevalence − 1` поруч із headline-метрикою.
- **NaN мовчки став порожнім рядком, а all-OOV документ — «вектором 0.1»** → `text_profile` рахує NaN/порожні/all-OOV окремо; `PooledTextEmbedding` дає нульовий вектор + явний `all_oov`-прапорець, не магічну константу.
- **Pretrained-енкодер нормалізують статистиками свого train** → для transfer learning брати константи передтренування ваг (ImageNet mean/std, `weights.transforms()`), не свої.
- **Відсутня папка класу мовчки зсуває всі індекси міток** (ImageFolder) → `folder_census` + `label_mapping_consistency` у readiness.
- **Argmax CV-кривої — шум фолдів** → правило одного SE (`one_se_rule`): найменший розмір у межах 1 SE від найкращого скору.
- **Імпутер обрано «за звичкою»** → `imputation_quality_probe`: замаскувати відомі значення й виміряти відновлення (так само обирати ранг `lowrank_impute`).
