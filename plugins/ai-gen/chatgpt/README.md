# ai-gen у Custom GPT (ChatGPT)

Той самий агент-ментор генеративного AI (клас моделі → модель → архітектура → оцінювання →
розгортання → приклад), що й у Claude, запакований для **Custom GPT** у ChatGPT. Інші способи
розгортання — у кореневому [README.md](../README.md).

Пакет — це два файли, що йдуть у пару:

- **[gpt_instructions.md](gpt_instructions.md)** — вставляється в поле «Instructions» Custom
  GPT (стиснута інструкція: bootstrap, обов'язкові правила, 6 етапів, handoff, обмеження
  ChatGPT-версії).
- **`dist/ai_gen_knowledge.zip`** — методологічний зміст (усі 6 `SKILL.md` +
  `references/*.md`), який GPT розпаковує собі через Code Interpreter і читає на вимогу.
  Будується скриптом **[build_gpt_package.ps1](build_gpt_package.ps1)**.

Чому саме так: поле Instructions має практичний ліміт **8000 UTF-8 байтів** (перевірте живий
ліміт у редакторі GPT на момент встановлення), тому Instructions несуть лише **процедуру**
(що робити і в якому порядку), а Knowledge-архів — **зміст**, який GPT читає вибірково.

> **Ліміт рахується в БАЙТАХ, не в символах** — для української інструкції це принципово:
> кирилиця займає 2 байти на літеру. `build_gpt_package.ps1` міряє саме байти
> (`UTF8.GetByteCount`) і друкує поточне значення проти ліміту — перед вставкою в ChatGPT
> переконайтеся, що бачите `OK`.

## Встановлення (перший раз)

1. **Зібрати пакет** (шлях залежить від того, де ви зараз):

   ```
   # якщо ви в корені плагіна ai-gen\:
   powershell -ExecutionPolicy Bypass -File chatgpt\build_gpt_package.ps1

   # якщо ви вже всередині ai-gen\chatgpt\:
   powershell -ExecutionPolicy Bypass -File build_gpt_package.ps1
   ```

   Скрипт очищує `__pycache__`, збирає `dist/ai_gen_knowledge.zip` і друкує довжину
   `gpt_instructions.md` у UTF-8 байтах проти ліміту 8000. Якщо `Warning` — інструкцію треба
   скоротити **до** вставки в ChatGPT, інакше він її відхилить.

2. **ChatGPT → Explore GPTs → Create → вкладка Configure** (не Create-чат-режим).

3. **Name / Description** — довільні, напр. «AI Gen Mentor».

4. **Instructions** — відкрити [gpt_instructions.md](gpt_instructions.md), скопіювати весь
   текст, вставити в поле Instructions.

5. **Capabilities** — увімкнути **Code Interpreter & Data Analysis** (обов'язково: без нього
   GPT не зможе розпакувати архів і читати довідки). Web browsing — корисно ввімкнути: агент
   зможе перевіряти актуальний каталог моделей замість цитування з пам'яті. DALL·E не потрібен.

6. **Knowledge** — завантажити `dist/ai_gen_knowledge.zip`.

7. **Save → Create.**

8. **Перевірка:** у новому чаті написати «Розпакуй knowledge і перелічи файли в
   design-agent-architecture/references». GPT має відповісти списком `.md`-файлів. Якщо каже,
   що не бачить архів — переконатися, що Code Interpreter увімкнено і файл справді
   завантажений у Knowledge.

## Як формулювати завдання

Мінімум, щоб почати — чотири речення: (1) яке рішення підтримує система (не «використати
LLM», а що зміниться для користувача), (2) вхід і вихід (модальності, формат), (3) обмеження
(латентність / бюджет / приватність / де запускати), (4) які дані є для навчання чи оцінки.

> Шаблон: «Хочу систему, що [рішення]. Вхід — [дані/модальність], вихід — [формат]. Обмеження:
> [латентність / бюджет / приватність / локально чи хмара]. Для оцінки якості маю [дані /
> нічого]. Порадь клас моделі, конкретні моделі й архітектуру, обґрунтуй.»

**Стартові фрази за метою:**

| Мета | Фраза |
|---|---|
| Повний цикл | «Проаналізуй задачу, обґрунтуй клас моделі та сплануй систему» |
| Клас моделі | «LLM чи класична модель для [задача]? Обґрунтуй з альтернативою» |
| Вибір моделі | «Обери 2–4 кандидати з OpenRouter під [вимоги] і спосіб їх порівняти» |
| Архітектура | «Спроєктуй [RAG / агента з інструментами / multi-agent] з ризиками» |
| Пам'ять | «Спроєктуй коротко- і довготривалу пам'ять та векторну БД для [кейс]» |
| Оцінювання | «Як міряти якість [система] і побудувати eval-набір» |
| Оптимізація | «Здешеви по токенах/латентності без втрати якості — з чого почати» |
| Розгортання | «Як запустити локально в Docker і потім в [AWS/GCP/Azure]» |
| Приклад | «Побудуй робочий приклад: LangGraph + OpenRouter + Tavily + React 19» |
| Handoff | «Зроби handoff сесії» |

## Оновлення (коли скіли зміняться тут, у Claude)

### A. Змінився лише вміст `references/*.md`

Instructions **не чіпати**. Зібрати пакет заново (див. команду вище) → у ChatGPT → редагувати
GPT → Knowledge → видалити старий `ai_gen_knowledge.zip` → завантажити новий → Save.

### B. Змінився сам `SKILL.md` (новий етап, нове обов'язкове правило)

Синхронізувати й **gpt_instructions.md**. Таблиця відповідності:

| Змінили в `SKILL.md` | Правити в `gpt_instructions.md` |
|---|---|
| `plan-ai-solution` → «Межі роботи» / «Обов'язкові правила» будь-якого скіла | «Обов'язкові правила» |
| `plan-ai-solution` → «Початковий контракт» | «Старт сесії» (+ блок «Як формулювати завдання» у цьому README) |
| `plan-ai-solution` → «Оркестрація скілів» (порядок/склад етапів) | «Етапи» (один рядок-ланцюжок) + `plan-ai-solution/references/skill-router.md` |
| `plan-ai-solution` → «Handoff між сесіями» / `references/handoff.md` | «Handoff між сесіями» |
| Новий референс у будь-якому скілі | Зазвичай **нічого правити** — GPT сам читає SKILL.md розпакованого архіву |
| Новий скіл | Додати в `$skills` у `build_gpt_package.ps1`, у ланцюжок «Етапи», у роутер, в `agents/ai-gen-mentor.md` і в дерево README плагіна — і перевірити байт-бюджет. `tests/check_docs.py` (перевірка 2) валить збірку, якщо ці списки розійшлися |

Детальна маршрутизація живе не в інструкції, а в
`plan-ai-solution/references/skill-router.md` — інструкція обмежена 8000 **байтами**, а
референси їдуть в архів безкоштовно. Правити маршрути треба там.

Після правки `gpt_instructions.md`: перезапустити `build_gpt_package.ps1` (переконатися в
`OK`), вставити оновлений текст в Instructions, замінити Knowledge-архів як у випадку A.

## Відомі обмеження цієї версії

- **Немає файлової системи проєкту**: handoff генерується текстом у чат — збережіть його собі
  й вставте на початку наступної сесії (у Claude Code handoff живе файлом `HANDOFF.md` у
  проєкті автоматично).
- **Пісочниця Code Interpreter скидається між розмовами** — нічого, записаного GPT під час
  чату, не переживає нову розмову.
- **Немає субагентів** — усе послідовно.
- **Каталог моделей не «в пам'яті»**: без увімкненого Web browsing GPT не може перевірити
  актуальні моделі/ціни — він дасть процедуру перевірки, а не цифри; це очікувана поведінка.
