# Стартові фрази й як коректно питати (ChatGPT-версія)

Цей файл — для **налаштування Custom GPT** і для користувача, який із ним
працює. У конструкторі GPT є окреме поле **Conversation starters** (4 слоти) —
вставити туди тексти з розділу 1 дослівно.

Агент — **сеньйор Data Scientist**. EDA вважається виконаним: його зона
починається зі сплітів/фолдів/батчів і йде до звіту.

## 1. Conversation starters (вставити в поле конструктора)

Чотири слоти, по одному на найчастіший вхід. Свідомо сформульовані так, щоб
одразу задавати правильний формат розмови, а не «привіт, що вмієш».

```
Постав задачу: є дані, треба прогноз — з чого почати?
```

```
Обери модель під мою задачу й обґрунтуй, чому саме вона
```

```
Спроєктуй схему валідації: у мене групи/час у даних
```

```
Модель готова, але результат гірший за очікуваний — діагностуй
```

**Чому саме ці чотири.** Вони покривають чотири різні входи в пак і ведуть у
різні скіли: постановка (`ml-task-framing`), вибір сімейства
(`ml-model-selection`), валідація (`ml-validation-design`), діагностика
(`ml-overfitting-diagnosis` / оркестратор). П'ятий частий випадок — «яку
метрику» — свідомо не винесений: він майже завжди приходить усередині перших
двох.

## 2. Що дати агентові, щоб відповідь була конкретною

Агент **сам запитає бракуюче**, але розмова буде коротшою, якщо одразу дати
підпис задачі. Мінімальний шаблон — скопіювати й заповнити:

```
Задача: <класифікація / регресія / кластеризація / прогноз ряду / ранжування>
Ціль (target): <що саме прогнозуємо, з вікном і горизонтом>
Дані: n = <рядків>, d = <ознак>; типи: <числові / категорії / текст / час>
Баланс: <частка позитивного класу> АБО розподіл цілі: <нулі? хвости? цензура?>
Ціна помилок: <що гірше — FP чи FN, і в чому вимірюється>
Структура: <є групи? часовий порядок? повторні виміри того самого об'єкта?>
Обмеження: <інтерпретованість? латентність? регуляторика? бюджет на навчання?>
Що вже є: <базлайн? спліт? метрика? навчена модель?>
```

**Найважливіші три рядки** — «Баланс», «Ціна помилок», «Структура». Без них
будь-яка порада про метрику, поріг і валідацію буде вгадуванням: саме тут
найчастіше й ламаються реальні проєкти.

## 3. Формулювання: слабке проти сильного

| Слабке питання | Чому погано | Сильна форма |
|---|---|---|
| «Яка модель найкраща?» | немає задачі й обмежень — відповідь буде загальником | «Табличні дані, n=50k, d=40, змішані типи, потрібна інтерпретованість для регулятора. Яке сімейство і чому?» |
| «Підбери гіперпараметри» | невідомо, що оптимізуємо і на чому міряємо | «Метрика PR-AUC, 5-фолдова StratifiedKFold, бюджет 2 години. Grid чи Optuna і яка сітка?» |
| «Точність 0.97, це добре?» | без балансу класів accuracy нічого не означає | «Позитивних 2%, accuracy 0.97, recall 0.11. Яку метрику брати замість accuracy?» |
| «Модель перенавчилась, що робити?» | немає чисел — діагноз неможливий | «Train AUC 0.99, val 0.71, RandomForest, n=3000. Це перенавчання чи витік?» |
| «Як розбити дані?» | відповідь залежить від структури, якої не назвали | «Кожен клієнт має 5-20 записів у різні дати. Як спліт, щоб не було витоку?» |
| «Порахуй VIF» | без мети незрозуміло, чи взагалі треба втручатися | «Хочу інтерпретувати коефіцієнти. VIF 3-12 на 8 ознаках — що прибрати?» |

Правило одним рядком: **числа + структура даних + що робитимете з прогнозом**.

## 4. Чого від агента чекати у відповіді

- **Питання назустріч**, якщо бракує критичного — це нормальна поведінка, не
  ухиляння.
- **Процедуру рішення**, а не лекцію: чому саме ця метрика/схема/сімейство.
- **Прогін скрипта** на ваших даних, де це можливо (потрібен увімкнений Code
  Interpreter — див. `README.md`).
- **Розкид, а не одне число**: метрика ± σ по фолдах; поріг як плато, не як
  число з чотирма знаками.
- **Позначки тверджень**: факт (пораховано) / інтерпретація / припущення.

## 5. Типові пастки самої ChatGPT-версії

- **Пісочниця скидається між чатами.** Усе важливе (конфіг, seed, числа) має
  потрапити у відповідь, а не лишитись у стані сесії. Довгий проєкт вести в
  одному чаті.
- **Дані треба завантажити в чат**, щоб агент їх бачив; без файлу він працює
  лише з вашим описом.
- **Версії бібліотек у пісочниці можуть бути старішими** за ті, під які
  писалися довідки пака (`references/api-2026.md` дато-штамповані). Якщо код
  падає — попросіть перевірити `sklearn.__version__` і адаптувати виклик.
