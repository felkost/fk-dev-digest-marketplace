# Custom GPT: встановлення і оновлення

Пакує **всі 22 скіли пака** в один Custom GPT для звичайного інтерфейсу
ChatGPT (актуальний список — масив `$skills` у `build_gpt_package.ps1`).
Це «Спосіб 3» з кореневого README; нативний OpenAI-агент формат
(`agents/openai.yaml`, виклик `$skill`) — окремий спосіб і в цей пакет не входить.

Персона GPT — **сеньйор Data Scientist**: EDA вважається виконаним, зона
відповідальності починається зі сплітів/фолдів/батчів і йде до фінального звіту.
Стартові фрази й шаблон коректного запиту — **`conversation-starters.md`**.

## Встановлення (один раз)

1. Зібрати пакет:

   ```
   powershell -ExecutionPolicy Bypass -File chatgpt\build_gpt_package.ps1
   ```

   Скрипт створює `dist/ml_advisor_knowledge.zip` і перевіряє, що
   `gpt_instructions.md` вкладається у **8000 байтів UTF-8** (ліміт ChatGPT
   рахується в байтах; кирилиця — 2 байти на літеру).

2. ChatGPT → Explore GPTs → **Create** → Configure:
   - **Instructions** ← вміст `chatgpt/gpt_instructions.md`;
   - **Capabilities** ← увімкнути *Code Interpreter & Data Analysis*
     (обов'язково: bootstrap розпаковує zip кодом); *Web browsing* — за бажанням
     (для фетчу PDF-доказів з чарта UDR; без нього GPT цитує з локального
     `udr-index.md` і `leemis-navigation.md`);
   - **Knowledge** ← завантажити `dist/ml_advisor_knowledge.zip`;
   - **Conversation starters** ← чотири фрази з `conversation-starters.md`
     (розділ 1), дослівно. Вони ведуть у чотири різні входи пака — постановка,
     вибір моделі, валідація, діагностика — замість беззмістовного «привіт,
     що ти вмієш».

3. Перевірка: спитати «яку метрику взяти, якщо позитивних 2%?» — GPT має
   розпакувати zip, прочитати `ml-metric-choice/SKILL.md` і відповісти
   процедурою (питання про ціну FP/FN), а не лекцією.

4. Друга перевірка — на персону: спитати навмисно загально («побудуй модель»).
   Правильна поведінка — **запитати бракуюче** (задача, n/d, баланс, ціна
   FP/FN, групи/час, обмеження), а не почати вгадувати архітектуру.

## Оновлення (коли скіли змінилися тут, у Claude-версії)

| Що змінилось | Що робити |
|---|---|
| `*/SKILL.md`, `*/references/*`, `*/scripts/*` | перезібрати `build_gpt_package.ps1`, перезалити zip у Knowledge (старий видалити) |
| `chatgpt/gpt_instructions.md` | скопіювати в поле Instructions заново |
| Додався новий скіл | додати в `$skills` у `build_gpt_package.ps1`, перезібрати. **Bootstrap правити не треба** — він підхоплює `*/scripts` через `glob` (з 2026-07-18; раніше був ручний список) |

Порада: тримайте Instructions мінімальними (маршрутизація + інваріанти), а весь
зміст — у Knowledge. Тоді оновлення зазвичай зводиться до перезаливки zip.

## Відомі обмеження ChatGPT-версії

- **Пісочниця скидається між розмовами** — жодного стану «на потім»; усе важливе
  має потрапити у відповідь.
- **Немає субагентів** — паралельні прогони скриптів виконуються послідовно.
- **sklearn у пісочниці може бути старішим** за локальний: скрипти пака свідомо
  не залежать від sklearn ≥ 1.9 (перебір порогів і GoF рахують самі), але
  `TunedThresholdClassifierCV` в порадах потребує ≥ 1.5.
- **Мережа**: чарт UDR має неповний ланцюг сертифікатів; при ввімкненому
  browsing фетчити `requests.get(..., verify=False)` лише для
  `www.math.wm.edu` — або покладатися на локальний індекс.
