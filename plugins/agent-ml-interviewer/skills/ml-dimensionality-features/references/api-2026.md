## Відбір: filter / embedded / wrapper

```python
from sklearn.feature_selection import (GenericUnivariateSelect, mutual_info_classif,
                                       SelectFromModel, RFECV, SequentialFeatureSelector)
from sklearn.pipeline import Pipeline

# filter (канон нотбука курсу):
sel = GenericUnivariateSelect(score_func=mutual_info_classif, mode="k_best", param=6)

# embedded L1 (стійка гілка нотбука):
from sklearn.linear_model import LogisticRegression
l1 = SelectFromModel(LogisticRegression(penalty="l1", solver="liblinear",
                                        class_weight="balanced", max_iter=500))

# embedded деревна з порогом (ТІЛЬКИ після канарок):
rf_sel = SelectFromModel(rf_fitted, threshold=0.1)          # або "median", "1.25*mean"

# wrapper (дорогий; як уточнення):
RFECV(estimator, step=1, cv=5, scoring="roc_auc", n_jobs=-1)
```

**k теж тюниться, і тільки в Pipeline** — сітка нотбука тюнить селектор разом
із моделлю: `{'selector__param': np.arange(4,10), 'rf__max_depth': ...}`.
Селектор поза Pipeline = витік (бачив валідаційні фолди).

`mutual_info_classif` ловить нелінійні залежності (кореляція — ні), але
одномірний: **XOR-пастку** (ознаки інформативні лише разом) не бачить ніхто з
філтрів — фінал звіряти з embedded/wrapper.

## Важливості: шість методів + канарки

```python
model.feature_importances_                      # 1. вбудована деревна (зміщена!)
np.abs(linear.coef_)                            # 2. коефіцієнти (після скейлінгу)
from sklearn.inspection import permutation_importance
permutation_importance(m, Xval, yval, n_repeats=10, n_jobs=-1)   # 3. чесніша
# 4. omission: перефіт без ознаки, дельта score
pca.components_                                 # 5. навантаження PCA (без цілі!)
# 6. SHAP: import shap; shap.Explainer(model)(X)   [окремий пакет]
```

Канарковий аудит перед будь-яким відбором за важливістю:

```
python ml-dimensionality-features/scripts/noise_sanity_check.py --csv d.csv --target y [--model l1] [--permutation]
```

Червоні прапори: канарка в топі; CV зросла після сміття (fnlwgt-сценарій).
Нюанс, вшитий у самотест: з redundant-ознаками L1 **законно** занулює частину
інформативних на користь їхньої комбінації — це близнюкова нестабільність
(`ml-linear-regularization`), а не провал канарок.

## PCA / LDA

```python
from sklearn.decomposition import PCA
from sklearn.discriminant_analysis import LinearDiscriminantAnalysis
from sklearn.preprocessing import StandardScaler

pipe = Pipeline([("sc", StandardScaler()),           # PCA чутливий до масштабу
                 ("pca", PCA(n_components=0.95))])   # частка дисперсії -> авто-k
pipe.fit(X); pipe["pca"].n_components_               # скільки компонент дали 95%
pipe["pca"].explained_variance_ratio_.cumsum()       # scree/кумулята для звіту

LinearDiscriminantAnalysis(n_components=2)           # ЖОРСТКО <= n_classes - 1
```

Перевірка користі — патерн нотбука: той самий `DecisionTreeClassifier` до/після
зниження, порівняти `roc_auc_score`.

## Пастки

| Пастка | Захист |
|---|---|
| селектор/скейлер зафічені до спліту | Pipeline під CV; `selector__param` у сітці |
| відбір за `feature_importances_` без аудиту | канарки; permutation |
| PCA без скейлінгу | StandardScaler у Pipeline |
| `LDA(n_components > класів-1)` | математична межа |
| t-SNE-координати як ознаки | тільки очі |
| `n_iter` у TSNE | `max_iter` (n_iter застарів у 1.5+) |
| «MI=0 ⇒ викинути» | XOR: перевірити взаємодію моделлю |
| SHAP на сирому пайплайні з горою колонок | спершу канарки й прунінг — дешевше |
