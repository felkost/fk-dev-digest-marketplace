## VIF правильно

```python
import statsmodels.api as sm
from statsmodels.stats.outliers_influence import variance_inflation_factor

Xc = sm.add_constant(X, has_constant="add")          # ОБОВ'ЯЗКОВО
vifs = {name: variance_inflation_factor(Xc, j + 1)   # j+1: пропускаємо константу
        for j, name in enumerate(feature_names)}
```

Без `add_constant` на даних із ненульовим середнім числа тихо хибні (виміряно:
17.6 замість 1.0 на незалежних ознаках). Готовий аудит із прунінгом:

```
python ml-linear-regularization/scripts/collinearity_report.py --csv d.csv --features a,b,c
```

## Регуляризовані моделі

```python
from sklearn.linear_model import Ridge, Lasso, ElasticNet, LogisticRegression
from sklearn.preprocessing import StandardScaler, PowerTransformer
from sklearn.pipeline import Pipeline

pipe = Pipeline([
    ("sc", StandardScaler()),          # або PowerTransformer("yeo-johnson") як у нотбуку
    ("m", Ridge(alpha=1.0)),
])
```

| Модель | Параметр сили | Сітка |
|---|---|---|
| Ridge / Lasso / ElasticNet | `alpha` (більший = сильніша) | `np.logspace(-4, 2, 13)` |
| ElasticNet | + `l1_ratio` | {0.1, 0.5, 0.9} |
| LogisticRegression | **C = 1/λ** (менший = сильніша!) | `np.logspace(-3, 1.5, 10)` — сітка нотбука |

Швидкі CV-версії з вбудованим підбором: `RidgeCV(alphas=...)`,
`LassoCV(cv=5)`, `ElasticNetCV(l1_ratio=[...], cv=5)`,
`LogisticRegressionCV(Cs=..., cv=5, scoring="roc_auc")`.

**Три живі `FutureWarning` на `LogisticRegressionCV` без явних аргументів**
(перевірено викликом `LogisticRegressionCV(cv=3).fit(...)` на 1.9.0 — усі три
спрацьовують одночасно, якщо не задати відповідні параметри):

1. **`scoring`**: *«The default value of the parameter 'scoring' will change
   from None, i.e. accuracy, to 'neg_log_loss' in version 1.11»* — задавайте
   `scoring=` явно (`"neg_log_loss"` для нової поведінки, `"accuracy"` щоб
   лишити стару).
2. **`l1_ratios`**: *«The default value for l1_ratios will change from None
   to (0.0,) in version 1.10»* — задавайте `l1_ratios=(0.0,)` явно, якщо
   `penalty="elasticnet"` не використовується.
3. **`use_legacy_attributes`** (нове): *«The fitted attributes of
   LogisticRegressionCV will be simplified in scikit-learn 1.10 to remove
   redundancy»* — обрати `use_legacy_attributes=False` вже зараз, щоб код не
   тримався на атрибутах, які зникнуть.

`lasso_path`/`enet_path`: **`n_alphas` підтверджено deprecated у 1.9** (живий
ворнінг: *«'n_alphas' was deprecated in 1.9 and will be removed in 1.11.
'alphas' now accepts an integer value which removes the need to pass
'n_alphas'»*) — тобто заміна не «інший параметр», а **той самий `alphas`
тепер приймає ціле число замість масиву** (перевірено: `alphas=15` дає рівно
15 значень α, так само як раніше `n_alphas=15`):

```python
lasso_path(X, y, alphas=100)      # було: n_alphas=100 -- тепер deprecated
lasso_path(X, y, alphas=my_grid)  # масив конкретних α, як і раніше
```

## Сумісність solver / penalty (LogisticRegression, не змінилось між 1.6 і 1.9)

| solver | l2 | l1 | elasticnet | Примітка |
|---|---|---|---|---|
| `lbfgs` (дефолт) | ✓ | — | — | ставте `max_iter=1000+` |
| `liblinear` | ✓ | ✓ | — | малі дані; нотбук курсу: `solver='liblinear', penalty='l1'` |
| `saga` | ✓ | ✓ | ✓ | великі дані; єдиний з elasticnet |

`class_weight="balanced"` — перший хід при дисбалансі (канон нотбука).

## statsmodels для інференсу

Коли потрібні p-values, довірчі інтервали, діагностика залишків:

```python
import statsmodels.api as sm
model = sm.OLS(y, sm.add_constant(X)).fit()
model.summary()                                   # коефіцієнти, SE, CI, F, R²
model.get_robustcov_results("HC3").summary()      # гетероскедастичність-стійкі SE
```

sklearn — для прогнозу і пайплайнів; statsmodels — для звіту «чому саме ці
коефіцієнти». Не змішуйте: у sklearn немає SE коефіцієнтів.

## Діагностика залишків (умови Гаусса-Маркова)

```python
resid = y - model.predict(...)
# гомоскедастичність: резид проти прогнозу — «воронка» = порушення умови 2
# нормальність: sm.qqplot(resid, line="45")
# автокореляція (часові): sm.stats.acorr_ljungbox(resid, lags=[10])
```

## Пастки

| Пастка | Захист |
|---|---|
| VIF без константи | `sm.add_constant`; скрипт це вшив |
| регуляризація без скейлінгу | scaler у Pipeline перед моделлю |
| scaler зафічений до спліту | Pipeline усередині CV (→ `ml-search-strategy`) |
| лінійна сітка по C/alpha | logspace |
| плутати напрям C і alpha | C = 1/λ: менший C — СИЛЬНІША регуляризація |
| `penalty='l1'` з дефолтним lbfgs | liblinear або saga |
| ворнінг збіжності проігноровано | `max_iter=1000+`, дивитись ConvergenceWarning |
| p-values зі sklearn-коефіцієнтів «на око» | statsmodels OLS/GLM summary |
| `LogisticRegressionCV()` без `scoring`/`l1_ratios`/`use_legacy_attributes` | 3 `FutureWarning` одразу (1.9); задавайте всі три явно |
| `lasso_path(..., n_alphas=N)` у новому коді | `alphas=N` (ціле число, не масив) |
