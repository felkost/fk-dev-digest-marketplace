## Сплітери: точні сигнатури

| Сплітер | Параметри (1.9.0) |
|---|---|
| `KFold` | `n_splits, shuffle, random_state` |

| `GroupKFold` | `n_splits, shuffle, random_state` |
| `StratifiedGroupKFold` | `n_splits, shuffle, random_state` |
| `TimeSeriesSplit` | `n_splits, max_train_size, test_size, gap` |
| `RepeatedStratifiedKFold` | `n_splits, n_repeats, random_state` |

**`GroupKFold` отримав `shuffle`/`random_state`** — у 1.6 їх не було, детермінований
порядок груп був єдиним варіантом. Тепер групи можна перемішувати між прогонами.

## TimeSeriesSplit: rolling проти expanding

Ключове: `max_train_size` перетворює expanding-вікно на **rolling**.

```python
from sklearn.model_selection import TimeSeriesSplit

# expanding: train росте від фолда до фолда (дефолт)
cv = TimeSeriesSplit(n_splits=5, test_size=30, gap=2)

# rolling: train фіксованої довжини, «забуває» старе
cv = TimeSeriesSplit(n_splits=5, test_size=30, gap=2, max_train_size=365)
```

- `test_size` — довжина кожного тестового вікна в спостереженнях (= горизонт).
- `gap` — скільки спостережень викинути між train і test (лаг даних, embargo).
- `max_train_size` — верхня межа train; без нього вікно розширюване.

## Групи + час: готового сплітера немає

У sklearn немає сплітера, який одночасно тримає групи цілими й зберігає часовий
порядок. Пишеться руками: відібрати групи у тестовий фолд, а потім обрізати train
за часом початку тесту (`train_mask &= час < min(час тесту)`). Порушення другої
частини — типовий тихий витік.

## Розкид, а не одне число

```python
from sklearn.model_selection import cross_validate
r = cross_validate(pipe, X, y, cv=cv, scoring="roc_auc")
r["test_score"].mean(), r["test_score"].std()   # звітувати ОБИДВА
```

`cross_validate` повертає `fit_time`, `score_time`, `test_score`;
`return_train_score=True` додає `train_score` (потрібен для діагностики розриву —
див. `ml-overfitting-diagnosis`).

## Вкладена CV

```python
inner = StratifiedKFold(5, shuffle=True, random_state=0)
outer = StratifiedKFold(5, shuffle=True, random_state=1)
search = GridSearchCV(pipe, grid, cv=inner, scoring="average_precision")
scores = cross_val_score(search, X, y, cv=outer, scoring="average_precision")
```

Зовнішній `cross_val_score` над `GridSearchCV` — і є вкладена CV. Число з неї —
оцінка **процедури**, а не конкретної моделі; фінальну модель після цього
навчають на всіх даних із найкращими параметрами.

## Adversarial validation (перевірка порівнянності наборів)

```python
import numpy as np, pandas as pd
from sklearn.ensemble import HistGradientBoostingClassifier
from sklearn.model_selection import cross_val_score

Z = pd.concat([X_train, X_test], ignore_index=True)
is_test = np.r_[np.zeros(len(X_train)), np.ones(len(X_test))]
auc = cross_val_score(HistGradientBoostingClassifier(), Z, is_test,
                      cv=5, scoring="roc_auc").mean()
# ~0.5 -> набори нерозрізненні; >0.6 -> дрейф, offline-оцінка не перенесеться
```

Ціль тут не потрібна — працює і на неозначених даних.

## Poisson bootstrap для довірчих меж

```python
rng = np.random.default_rng(0)
w = rng.poisson(1.0, size=len(y_true))          # вага кожного спостереження
# метрика з вагами, B разів -> розподіл -> np.quantile(dist, 0.01) як нижня межа
```

`Binomial(n, 1/n) ≈ Poisson(1)` при n ≫ 100: у виразі немає n, тому ітерації
незалежні й тривіально паралеляться — на відміну від класичного мультиноміального
bootstrap.
