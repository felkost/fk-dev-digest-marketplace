## Криві навчання і валідації

```python
from sklearn.model_selection import learning_curve, validation_curve, cross_validate

# score від розміру train: розрізняє "мало даних" від "не та модель"
sizes, tr, te = learning_curve(est, X, y, cv=5, train_sizes=np.linspace(.1, 1., 8),
                               scoring="roc_auc", n_jobs=-1)

# score від ОДНОГО гіперпараметра: видно зону over/underfit
tr, te = validation_curve(est, X, y, param_name="max_depth",
                          param_range=range(1, 16), cv=5, scoring="roc_auc")

# train і test score разом — сировина для розриву
res = cross_validate(est, X, y, cv=5, scoring="roc_auc",
                     return_train_score=True, n_jobs=-1)
gap = (res["train_score"].mean() - res["test_score"].mean()) / res["train_score"].mean()
```

`return_train_score=False` за замовчуванням — **увімкніть явно**, без train-score
розрив нерахований.

## Швидкий вердикт без коду

## Пастки

| Пастка | Наслідок | Захист |
|---|---|---|
| препроцесор зафічено до спліту | витік → test оптимістичний | `Pipeline` усередині CV (→ `ml-search-strategy`) |
| дублікати поперек спліту | test бачив train | дедуплікація до спліту |
| `cross_val_score` без `return_train_score` | розрив нерахований | `cross_validate(..., return_train_score=True)` |
| argmax по CV-сітці | шум фолдів | правило 1 SE |
| CV зросла після додавання ознак-канарок | протокольний витік | `noise_sanity_check.py` |
| порівняння метрик різних шкал (R² проти RMSE) | розрив беззмістовний | один вердикт = одна метрика |
