## Тотожність J = 2·BA − 1

Це те, що робить Youden's J дефолтом `TunedThresholdClassifierCV`.

```
J   = TPR − FPR
    = Sens − (1 − Spec)
    = Sens + Spec − 1

BA  = (Sens + Spec) / 2
```

Отже `Sens + Spec = 2·BA`, і підстановка дає **J = 2·BA − 1**. Перетворення строго
монотонне зростаюче, тому `argmax_t J(t) = argmax_t BA(t)`: максимізація balanced
accuracy **тотожна** критерію Юдена. Тому `TunedThresholdClassifierCV(est)` зі
своїм дефолтом `scoring='balanced_accuracy'` реалізує саме Youden's J.

Слайд Лекції 2 підводить до цього з іншого боку: «тепер зрозуміло, чому BA добре
працює на незбалансованих даних» — бо BA це лінійна перешкалована J, а J
за побудовою не залежить від часток класів (обидва доданки — умовні ймовірності).

## Чотири критерії

| Критерій | Цільова функція | Умова оптимуму на ROC |
|---|---|---|
| closest-to-(0,1) | min √((1−TPR)² + FPR²) | ROC′(x) = x |
| Youden's J | max (TPR − FPR) | ROC′(x) = 1 |
| min\|Sens−Spec\| | min \|(1−FPR) − TPR\| | перетин ROC з антидіагоналлю |
| доменне обмеження | max TPR за умови Spec ≥ s₀ (або Precision ≥ p₀) | межа допустимої області |

Умови `ROC′(x) = x` і `ROC′(x) = 1` виводяться прирівнюванням похідної цільової
функції до нуля вздовж кривої ROC.

**J ∈ [0, 1]** для розумного класифікатора. J < 0 означає, що мітки переплутано
місцями.

## Модельна задача: замкнені форми

Лекція 2 будує задачу, у якій усе рахується аналітично. Бали класу 0 мають
густину `f₀(x) = 2 − 2x`, класу 1 — `f₁(x) = 2x`, обидві на [0, 1].
**Апріорні ймовірності рівні** — без цієї умови формула для Precision інша.

Правило: передбачаємо 1, якщо `x ≥ t`.

```
Recall     R(t) = ∫ₜ¹ 2x dx = 1 − t²

FPR        F(t) = ∫ₜ¹ (2 − 2x) dx = 2(1−t) − (1−t²) = (1−t)²

Precision  P(t) = R / (R + FPR)                       [рівні апріорні]
                = (1−t²) / [(1−t²) + (1−t)²]
                = (1−t)(1+t) / [(1−t)(1+t) + (1−t)²]
                = (1+t) / [(1+t) + (1−t)]
                = (1+t) / 2
```

Решта метрик як функції порогу (Лекція 2):

```
F1(t)   = 2PR/(P+R) = (1 − t²) / (3/2 − t)
MCC(t)  = √(t(1−t))
BA(t)   = Accuracy(t) = (1 + 2t − 2t²) / 2
κ(t)    = 2t(1−t)
```

Максимуми і точки, де вони досягаються:

| Метрика | Максимум | при t |
|---|---|---|
| Precision | 1 | t = 1 |
| Recall | 1 | t = 0 |
| **F1** | **3 − √5 ≈ 0.7639** | **(3−√5)/2 ≈ 0.3820** |
| MCC | ½ | ½ |
| BA | ¾ | ½ |
| κ | ½ | ½ |

Зверніть увагу: **різні метрики максимізуються в різних точках**. Це і є причина,
чому «підібрати поріг» без відповіді на питання «під яку метрику» — беззмістовно.

## Максимізація F1 по порогу

```
F1(t) = (1 − t²) / (3/2 − t)

Нехай u = 1 − t²,  v = 3/2 − t,  тоді u′ = −2t,  v′ = −1.

dF1/dt = (u′v − uv′)/v² = 0   ⟹   u′v − uv′ = 0
       (−2t)(3/2 − t) − (1 − t²)(−1) = 0
       −3t + 2t² + 1 − t² = 0
       t² − 3t + 1 = 0
       t = (3 ± √5)/2
```

Корінь у [0,1]: **t\* = (3 − √5)/2 ≈ 0.381966**.

Значення в максимумі. З `t² = 3t − 1` маємо `1 − t² = 2 − 3t`, тож

```
F1(t*) = (2 − 3t)/(3/2 − t) = 2(2 − 3t)/(3 − 2t)
```

Підстановка `2t = 3 − √5` дає `3 − 2t = √5` і `2 − 3t = (3√5 − 5)/2`, отже

```
F1(t*) = (3√5 − 5)/√5 = 3 − 5/√5 = 3 − √5 ≈ 0.763932
```

## AUC-ROC і AUC-PRC аналітично

**AUC-ROC** через тотожність `AUC = P(ξ₁ > ξ₀)`:

```
AUC = ∫₀¹ f₁(y)·F₀(y) dy,   де F₀(y) = ∫₀^y (2−2x)dx = 2y − y²
    = ∫₀¹ 2y(2y − y²) dy
    = ∫₀¹ (4y² − 2y³) dy
    = 4/3 − 1/2 = 5/6 ≈ 0.8333
```

**AUC-PRC** через інверсію. З `R = 1 − t²` маємо `t = √(1−R)`, а `P = (1+t)/2`, тож

```
P(R) = (1 + √(1−R)) / 2

AUC-PRC = ∫₀¹ ½(1 + √(1−R)) dR = ½(1 + 2/3) = 5/6
```

Тут AUC-ROC = AUC-PRC = 5/6. **Це збіг саме цієї задачі й рівних апріорних
ймовірностей — у загальному випадку вони не рівні** (Лекція 2 наголошує це окремо).

## Плоскість оптимуму

Виміряно на аналітичній формі `F1(t) = (1−t²)/(3/2−t)`:

| t | F1(t) | втрата від максимуму |
|---|---|---|
| 0.232 | 0.746188 | 0.017744 |
| 0.282 | 0.755722 | 0.008210 |
| 0.332 | 0.761792 | 0.002140 |
| **0.382** | **0.763932** | **0** |
| 0.432 | 0.761591 | 0.002341 |
| 0.482 | 0.754109 | 0.009823 |
| 0.532 | 0.740689 | 0.023243 |

Відхилення ±0.05 коштує 0.2% F1, ±0.10 — близько 1%. Наслідок для практики:
**оцінка `t` погано ідентифікована навіть тоді, коли сам F1 оцінено точно.**
Самотест це відтворює — при зростанні n від 4 000 до 400 000 F1 сходиться до 5-го
знака, а `t̂` гуляє в 0.378–0.386. Подавайте поріг як плато, а не як число з
чотирма знаками.

## Gini = 2·AUC − 1

Коефіцієнт Джині як площа між кривою Лоренца і діагоналлю: `Gini ≡ A/(A+B)`.
Оскільки `A + B = ½`, маємо `Gini = 2A`. Перехід до AUC дає **Gini = 2·AUC − 1**,
що нормує випадковий класифікатор у 0, а ідеальний у 1; діапазон [−1, 1].

Джерело: Harel Lustiger, *The Gini Coefficient under Binary Classification
Problems* (2017); Corrado Gini (1884–1965); крива Лоренца — Max Lorenz (1905).
