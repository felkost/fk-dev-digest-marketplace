# API: підбір порогу

**Перевірено 2026-07-18 проти встановленої scikit-learn 1.9.0** (оновлено з
1.6.1 того самого дня; усе нижче знову перевірено інтроспекцією й живими
викликами на новій версії — не переказ release notes).

Це єдине місце у скілі, де є прив'язка до версій. Оновлення бібліотеки правиться тут.

## Мінімальні версії

| Що потрібно | Версія | Є в 1.9.0? |
|---|---|---|
| `TunedThresholdClassifierCV` | ≥ 1.5 | так |
| `FixedThresholdClassifier` | ≥ 1.5 | так |
| `sklearn.frozen.FrozenEstimator` | ≥ 1.6 | так |
| `CalibratedClassifierCV(ensemble=False)` | давно | так |
| `metric_at_thresholds` | ≥ 1.9 | **так** |
| `PrecisionRecallDisplay.from_cv_results` | ≥ 1.9 | **так** |
| `cohen_kappa_score(replace_undefined_by=...)` | ≥ 1.9 | **так** |

`scripts/threshold_report.py` і далі не залежить від 1.9 навмисно — перебір
порогів він рахує сам (numpy + pandas), і це та сама побудова, яку Лекція 2
виконує вручну. Але тепер `metric_at_thresholds` дає той самий результат одним
викликом sklearn — дивіться нижче.

## TunedThresholdClassifierCV

```python
from sklearn.model_selection import TunedThresholdClassifierCV
```

Сигнатура (1.9.0, перевірено інтроспекцією — не змінилась із 1.6):

```
estimator, scoring='balanced_accuracy', response_method='auto',
thresholds=100, cv=None, refit=True, n_jobs=None,
random_state=None, store_cv_results=False
```

| Параметр | Що робить | Порада |
|---|---|---|
| `scoring` | метрика, яку максимізуємо по порогу | **дефолт `'balanced_accuracy'` = Youden's J** (бо J = 2·BA − 1) |
| `cv` | схема внутрішньої CV | дефолт 5-fold stratified; `cv='prefit'` для вже навченої моделі — але тоді поріг зміщений, якщо це ті самі дані |
| `response_method` | `predict_proba` чи `decision_function` | `'auto'` майже завжди правильно |
| `thresholds` | скільки кандидатів перебрати | 100 достатньо; оптимум плоский |
| `store_cv_results` | зберегти криву | `True`, якщо хочете побачити плато |

Після `fit`: `best_threshold_`, `best_score_`, і (за `store_cv_results=True`)
`cv_results_`.

```python
from sklearn.metrics import make_scorer, fbeta_score

# Youden's J -- дефолт, аргументи не потрібні
TunedThresholdClassifierCV(est, cv=5)

# F-beta: beta=2 коли пропуск дорожчий, beta=0.5 коли хибна тривога дорожча
TunedThresholdClassifierCV(est, scoring=make_scorer(fbeta_score, beta=2), cv=5)

# Метрика для конкретного класу (напр. рідкісний клас позначено 0)
TunedThresholdClassifierCV(est, scoring=make_scorer(f1_score, pos_label=0))
```

## FixedThresholdClassifier

Коли поріг заданий ззовні (регламент, домовленість із бізнесом) і модель
перенавчати не можна:

```python
from sklearn.model_selection import FixedThresholdClassifier
from sklearn.frozen import FrozenEstimator

clf = FixedThresholdClassifier(FrozenEstimator(fitted_est), threshold=0.30)
```

`FrozenEstimator` **обов'язковий**, інакше `fit` перенавчить вкладену модель.

## Калібрування перед підбором

Поріг має сенс лише на монотонно осмислених балах. Для SVC і подібних:

```python
from sklearn.calibration import CalibratedClassifierCV
calibrated = CalibratedClassifierCV(SVC(), ensemble=False, cv=5)
```

**`SVC(probability=True)` формально задеприкейчено рівно у 1.9.0** — перевірено
живим `FutureWarning`: *«The `probability` parameter was deprecated in 1.9 and
will be removed in version 1.11. Use `CalibratedClassifierCV(SVC(),
ensemble=False)` instead…»*. `probA_` / `probB_` під той самий приціл.
Рекомендована заміна — саме `CalibratedClassifierCV(ensemble=False)`.

## Перебір порогів вручну

Ручний варіант (і те, що робить `threshold_report.py`):

```python
from sklearn.metrics import roc_curve, precision_recall_curve
fpr, tpr, thr = roc_curve(y_true, y_score)
J = tpr - fpr
t_youden = thr[J.argmax()]
```

Обережно: `roc_curve` повертає перший поріг `inf` (точка (0,0)) — у старих версіях
це було `max(score)+1`. Не інтерпретуйте його як реальний поріг.

**З 1.9 те саме одним викликом** (перевірено — повертає прямий tuple
`(значення_метрики, пороги)`, не іменований кортеж і не dict):

```python
from sklearn.metrics import metric_at_thresholds, f1_score

f1_values, thresholds = metric_at_thresholds(y_true, y_score, f1_score)
t_best = thresholds[f1_values.argmax()]
```

Сигнатура: `metric_at_thresholds(y_true, y_score, metric_func, *,
sample_weight=None, metric_params=None)` — `metric_func` приймає будь-яку
функцію виду `(y_true, y_pred) -> float`, включно з `fbeta_score` через
`functools.partial`.

## Що НЕ використовувати

| Застаріле / хибне | Заміна |
|---|---|
| `SVC(probability=True)` | `CalibratedClassifierCV(SVC(), ensemble=False)` |
| `(y_proba > 0.5).astype(int)` як дефолт | підбір порогу під метрику |
| підбір порогу на train | `TunedThresholdClassifierCV` з внутрішньою CV |
| `cv='prefit'` на тих самих даних, де вчили | окремий валідаційний набір |
