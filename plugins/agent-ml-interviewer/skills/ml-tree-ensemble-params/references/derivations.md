## Ентропія та Information Gain

```
E(D) = − Σᵢ pᵢ log₂ pᵢ
```

Межі: чистий вузол → E = −1·log₂1 = 0; рівномірний по k класах → E = log₂k
(максимум). Приріст інформації від розбиття за атрибутом V:

```
Gain(D, V) = E(D) − Σᵥ (Nᵥ/N) · E(Dᵥ)
```

## Приклад Надаля: повний рахунок

14 днів тренувань, атрибути Outlook / Temp / Humidity / Wind, ціль Play/Stay
(9 Play, 5 Stay).

```
E(D) = −(9/14)log₂(9/14) − (5/14)log₂(5/14) = 0.940
```

| Атрибут | Ентропії гілок | Gain |
|---|---|---|
| **Outlook** | Sunny 0.722, Overcast 0, Rain 0.722 | **0.424** ← корінь |
| Wind | Weak 0.764, Strong 0.970 | 0.102 |
| Humidity | High 0.863, Normal 0.985 | 0.016 |
| Temp | Hot 0.811, Mid 0.970, Cold 0.970 | 0.008 |

Overcast → E = 0: усі 4 «Overcast»-дні — Play; гілка стає листком одразу.

## Зсув IG: атрибут «Дата»

Додаємо атрибут «Дата» — 14 унікальних значень на 14 рядків. Кожна гілка
одноелементна, отже чиста, отже:

```
Gain(D, Дата) = E(D) − 0 = 0.940   — максимум можливого
```

IG обирає «Дату» — атрибут із нульовою прогнозною силою. Це загальний механізм:
**IG зміщений до багатозначних атрибутів** (ID у sklearn-практиці — той самий
випадок: одиничні листки, ідеальний train, нуль на test).

## SplitInfo і Gain Ratio

```
SplitInfo(D, V) = − Σᵥ (Nᵥ/N) log₂(Nᵥ/N)      — ентропія самого розбиття
GainRatio(D, V) = Gain(D, V) / SplitInfo(D, V)
```

| Атрибут | Gain | SplitInfo | GainRatio |
|---|---|---|---|
| **Outlook** | 0.424 | 1.577 | **0.269** ← знову корінь |
| Дата | 0.940 | 3.807 | 0.246 |
| Wind | 0.102 | ~0.93 | 0.110 |
| Humidity | 0.016 | 1.000 | 0.016 |
| Temp | 0.008 | ~1.56 | 0.005 |

Нормування на SplitInfo повертає «Дату» на місце. Висновок курсу: «Gain ratio
надійніший за Information Gain» (C4.5 використовує саме його).

## Gini і мапа критерій→алгоритм

```
Gini(D) = 1 − Σᵢ pᵢ²
```

| Критерій | Алгоритм |
|---|---|
| Information Gain | ID3, C4.5 (через Gain Ratio) |
| Gini | **CART** (= sklearn), IBM IntelligentMiner |

Raileanu & Stoffel, *Theoretical comparison between the Gini Index and
Information Gain criteria* (2004): вибір критерію змінює результат лише у ~2%
випадків; entropy дорожча через log. Звідси дефолт: gini, бюджет — на ємність.

## Процедура обрізання Л6-7

**Pre-pruning** — три стоп-умови росту: (а) мінімальна кількість об'єктів у
вузлі; (б) досягнута максимальна глибина; (в) приріст (Gain) нижче порога.
У sklearn: `min_samples_split`/`min_samples_leaf`, `max_depth`,
`min_impurity_decrease`.

**Post-pruning** — на відкладеній вибірці, конвенційно **½ розміру train**:
обхід внутрішніх вузлів (порядок — деталь реалізації); у кожному вузлі
порівнюються за accuracy на холдауті **чотири кандидати**: без змін / замінити
лівим піддеревом / правим / листком із мажоритарним класом холдауту в цьому
вузлі; якщо жоден об'єкт холдауту не доходить до вузла — листок із мажоритарним
класом train.

**Cost-complexity (CART/sklearn):** мінімізується `R_α(T) = R(T) + α·|leaves|`.
Зростання α стискає дерево; `cost_complexity_pruning_path` повертає скінченний
список критичних α — кандидатів для CV.

## Чому ансамбль б'є одну модель: біноміальний аргумент

Л13: n незалежних класифікаторів із помилкою ε, мажоритарне голосування
помиляється, коли помиляється більшість:

```
P(ансамбль помилився) = Σ_{k>n/2} C(n,k) ε^k (1−ε)^{n−k}
```

Опрацьований випадок: n=11, ε=0.25 →
`Σ_{k=6}^{11} C(11,k)·0.25^k·0.75^{11−k} = 0.034` — з 25% індивідуальної помилки
до 3.4% ансамблевої. Умови, на яких усе тримається: помилки **незалежні** і
ε < 0.5. Звідси вся механіка різноманіття: бутстреп, підпростори ознак.

## Бутстреп і 1/e ≈ 36.8% (OOB)

Ймовірність, що конкретний рядок НЕ потрапить у бутстреп-вибірку розміру n:

```
(1 − 1/n)ⁿ  →  e⁻¹ ≈ 0.368   при n→∞
```

Отже кожне дерево не бачить ~36.8% рядків — на них його можна чесно оцінити.
Усереднення по деревах дає **OOB-оцінку** (`oob_score=True`) — валідація без
окремого холдауту.

## Bagging / Pasting / Subspaces / Patches / RF

Порівняльна таблиця Л13:

| Метод | Рядки | Ознаки |
|---|---|---|
| Bagging | бутстреп (з поверненням) | усі |
| Pasting | випадкова підвибірка без повернення | усі |
| Random Subspaces | усі | випадкова підмножина |
| Random Patches | підвибірка | підмножина |
| **Random Forest** | **бутстреп** | **підмножина на кожен спліт** |

RF = bagging + random subspaces на рівні сплітів; `max_features` — і є та сама
підмножина, головна ручка декореляції дерев.
