# API: дерева та ансамблі

**Перевірено 2026-07-18 на scikit-learn 1.9.0** (оновлено з 1.6.1 того самого
дня; усе нижче — живі сигнатури й виклики, не переказ реліз-нотаток).

## Одне дерево

```python
from sklearn.tree import DecisionTreeClassifier, plot_tree

tree = DecisionTreeClassifier(criterion="gini",      # entropy ≈ те саме у ~98% випадків
                              max_depth=4, min_samples_leaf=10,
                              random_state=1)
```

Пост-обрізання за вартістю-складністю (механізм CART, аналог процедури Л6-7):

```python
path = DecisionTreeClassifier(random_state=1).cost_complexity_pruning_path(Xtr, ytr)
alphas = path.ccp_alphas[:-1]                        # останній стискає в корінь
# кандидати α -> CV -> найпростіше дерево в 1 SE від кращого
```

`plot_tree(tree, feature_names=..., filled=True)` — обов'язкова перевірка очима:
листки-мемуари видно одразу.

## Random Forest

```python
from sklearn.ensemble import RandomForestClassifier
rf = RandomForestClassifier(n_estimators=300, max_features="sqrt",
                            oob_score=True,          # безкоштовна оцінка на ~36.8% поза бутстрепом
                            class_weight="balanced", # при дисбалансі
                            n_jobs=-1, random_state=1)
rf.fit(Xtr, ytr); rf.oob_score_
```

Важливості: `rf.feature_importances_` (impurity — зміщена до багатокардинальних
і ділиться між близнюками) проти чеснішої:

```python
from sklearn.inspection import permutation_importance
pi = permutation_importance(rf, Xval, yval, n_repeats=10, random_state=1, n_jobs=-1)
```

## AdaBoost — що змінилось проти курсу

```python
from sklearn.ensemble import AdaBoostClassifier
from sklearn.tree import DecisionTreeClassifier

ada = AdaBoostClassifier(estimator=DecisionTreeClassifier(max_depth=1),
                         n_estimators=200, learning_rate=0.5, random_state=1)
```

| Було (курс/старі приклади) | 1.6.1 | **1.9.0 (перевірено щойно)** |
|---|---|---|
| `base_estimator=` | видалено → `estimator=` | те саме — `estimator=` |
| `algorithm="SAMME.R"` дефолт, SAMME.R швидший | deprecated, не діє; ворнінг | **параметра більше немає взагалі** |

**Це реальна зламна зміна між 1.6 і 1.9, не просто «ще одне попередження».**
На 1.6 виклик `AdaBoostClassifier(algorithm="SAMME")` мовчки ігнорував
аргумент (з ворнінгом). На 1.9 той самий виклик кидає:

```
TypeError: AdaBoostClassifier.__init__() got an unexpected keyword argument 'algorithm'
```

Якщо в проєкті десь лишився `algorithm=` (з міграції старого коду або з
пам'яті LLM, навченої на курсі 2024 року) — оновлення sklearn до 1.9 його
зламає. Курсова таблиця SAMME проти SAMME.R (Л13) — тепер суто історична
довідка: у 1.9 працює лише алгоритм SAMME, і про вибір між ними більше не
йдеться.

## Градієнтний бустинг

Дефолтний вибір — гістограмний:

```python
from sklearn.ensemble import HistGradientBoostingClassifier
hgb = HistGradientBoostingClassifier(
    learning_rate=0.1, max_iter=300,
    early_stopping=True, validation_fraction=0.1, n_iter_no_change=20,
    max_leaf_nodes=31, l2_regularization=0.0,
    categorical_features="from_dtype",     # pandas category → нативні категорії, без one-hot
    random_state=1)
```

Класичний `GradientBoosting*` — для малих даних; `subsample=0.8` дає
стохастичний бустинг. **Параметр `criterion` у 1.9 видає живий `FutureWarning`**
(дослівно: *«The parameter `criterion` is deprecated and will be removed in
1.11. It has no effect. Leave it to its default value…»*) — `friedman_mse` ≡
`squared_error` за результатом; не задавайте `criterion` у новому коді.

Зв'язка lr×iter: типові пари (0.1, 300) ≈ (0.03, 1000); нижчий lr + рання
зупинка — типово краща генералізація за той самий бюджет.

## Пастки

| Пастка | Захист |
|---|---|
| `base_estimator=` в AdaBoost | `estimator=` (видалено ще в 1.6) |
| `algorithm=` в AdaBoost на sklearn ≥1.9 | **TypeError** — параметр видалено повністю, приберіть з коду |
| one-hot категорій у дерева | `categorical_features` (HGB) / нативні категорії |
| тюнінг `n_estimators` RF сіткою | 300 фіксовано; крутити `max_features` |
| impurity-важливість як відбір | permutation + канарки (`ml-dimensionality-features`) |
| `criterion="friedman_mse"` у новому коді | `FutureWarning`, видалять у 1.11; не задавати |
