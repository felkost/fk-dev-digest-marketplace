## scipy.stats: фіт і перевірка

Мапа імен «розподіл → scipy»:

| Розподіл | scipy.stats | Примітка |
|---|---|---|
| Poisson | `poisson` | дискретний; MLE = вибіркове середнє |
| NegBin (Pascal) | `nbinom` | параметризація (n, p), не (mean, overdispersion)! |
| Gamma | `gamma` | `floc=0` майже завжди обов'язковий |
| Lognormal | `lognorm` | `s`=σ лог-шкали; `scale`=exp(µ); `floc=0` |
| Weibull | `weibull_min` | не `weibull_max` і не `exponweib` |
| Exponential | `expon` | `scale`=1/λ |
| GEV | `genextreme` | знак shape протилежний конвенції Coles |
| GPD (хвіст) | `genpareto` | фітити на перевищеннях порога |
| Pareto | `pareto` | |
| Student t | `t` | оцінений df < 5 — маркер важких хвостів |
| Laplace | `laplace` | MLE локації = медіана |
| Beta | `beta` | support строго (0,1); 0/1 треба зсунути |
| von Mises | `vonmises` | циркулярний |

```python
from scipy import stats
# Неперервні: MLE-фіт; для додатних фіксуйте floc=0, інакше scipy
# «покращує» правдоподібність зсувом локації і параметри стають нечитабельними
params = stats.lognorm.fit(x, floc=0)
loglik = stats.lognorm.logpdf(x, *params).sum()
aic = 2 * len(params) - 2 * loglik          # порівнювати кандидатів
```

**Пастка GoF:** `stats.kstest(x, "lognorm", args=params)` з параметрами,
оціненими на тих самих даних, дає завищений p-value (ефект Ліллієфорса).
Валідний шлях — параметричний бутстреп (реалізовано в
`scripts/distribution_candidates.py`): симулювати з фітованого, **рефітити на
кожній симуляції**, порівняти KS-статистики.

**Пастка великих n:** тест нормальності (Shapiro, D'Agostino) на n ≥ 10⁴
відкидає все — читайте величину відхилення (QQ, ексцес), не p-value.

## sklearn: моделі за розподілом

```python
from sklearn.linear_model import (
    PoissonRegressor,      # лічильні, log-link
    GammaRegressor,        # додатні, CV ~ const, log-link
    TweedieRegressor,      # power=0 Normal, 1 Poisson, (1,2) compound, 2 Gamma, 3 InvGauss
    QuantileRegressor,     # pinball-втрата, будь-який розподіл
    HuberRegressor,        # симетричний шум з викидами
)
from sklearn.ensemble import HistGradientBoostingRegressor
# перевірено на 1.9.0: loss = "poisson" | "gamma" | "quantile" (+ squared/absolute)
HistGradientBoostingRegressor(loss="poisson")
HistGradientBoostingRegressor(loss="gamma")
HistGradientBoostingRegressor(loss="quantile", quantile=0.9)
```

**Експозиція в Poisson:** sklearn не має offset-параметра — передавайте
`sample_weight=exposure` і моделюйте rate = counts/exposure, або беріть
statsmodels GLM з `offset=np.log(exposure)` (канонічний шлях).

## sklearn.metrics: оцінювання тим самим розподілом

```python
from sklearn.metrics import (
    mean_poisson_deviance,   # лічильні
    mean_gamma_deviance,     # додатні
    mean_tweedie_deviance,   # power узгодити з моделлю
    d2_tweedie_score,        # аналог R² для deviance (1 - dev/dev_null)
    mean_pinball_loss,       # квантильна: alpha = той самий τ, що в моделі
    mean_squared_log_error,  # RMSE на log1p-шкалі; лише y >= 0
)
```

Правило узгодження: **метрика = та сама deviance, що й втрата навчання**,
на відкладеній вибірці. Оцінювати Poisson-модель через RMSE — значить знову
мовчки припустити гаусів шум, який ви щойно відкинули.

`d2_tweedie_score(power=...)` — узагальнення R²: 0 — не краще за
константу-передбачення, 1 — ідеально; працює для Poisson (power=1) і
Gamma (power=2).

## Survival (окремі пакети, теж не встановлені)

Цензурований час до події — `lifelines` (WeibullAFTFitter, KaplanMeier,

## Швидкі діагностики без бібліотек

```python
import numpy as np
vmr = x.var(ddof=1) / x.mean()                    # лічильні: 1 -> Poisson
zero_gap = (x == 0).mean() - np.exp(-x.mean())    # надлишок нулів проти Poisson
log_skew = stats.skew(np.log(x[x > 0]))           # ~0 -> lognormal
tail_ratio = np.quantile(x, .99) / np.quantile(x, .50)   # грубий маркер хвоста
```

## Що НЕ використовувати

| Хибне | Чому | Заміна |
|---|---|---|
| MSE на лічильних | V=µ вбудована гетероскедастичність | Poisson deviance |
| RMSE на цензурованому часі | зміщена підвибірка «завершених» | survival + C-index |
| `kstest` з фітованими параметрами | p-value завищений (Lilliefors) | параметричний бутстреп |
| Shapiro на n=10⁵ | завжди відкидає | QQ + величина відхилення |
| `lognorm.fit(x)` без `floc=0` | зсув локації, нечитабельні параметри | `fit(x, floc=0)` |
| середнє для Cauchy-подібних | не існує / не збігається | медіана, MAE, бутстреп-CI |
| MSLE на даних із нулями «як є» | log(0) | `log1p`-семантика вже в метриці; y<0 — заборонено |
