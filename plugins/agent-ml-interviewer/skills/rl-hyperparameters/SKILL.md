# Гіперпараметри навчання з підкріпленням

Три ручки табличного RL — γ, α, ε — і вибір алгоритму. Термінологія і позначення
— за курсом Жука (Sutton & Barto).

**Відповіді українською; ідентифікатори й код — англійською.**

## Не відповідає на

- **Пошук гіперпараметрів supervised-моделей** → `ml-search-strategy`.
- **Нейромережеві деталі DQN/policy-gradient** — поза паком; тут — коли їх
  обирати і які табличні основи вони успадковують.

## Крок 0 — постановка як MDP

Перш ніж тюнити: назвати стани, дії, винагороду, і чи задача **епізодична**
(T < ∞: гра, прохід лабіринту) чи **неперервна** (T = ∞: керування, трейдинг).
Віддача: `G_t = Σ_k γ^k R_{t+k+1}`; рекурсивно `G_t = R_{t+1} + γ·G_{t+1}`.
Марковість — усе, що впливає на майбутнє, має бути в стані; якщо ні —
розширювати стан, а не тюнити γ.

## Крок 1 — γ: горизонт, а не «ще один параметр»

| γ | Ефективний горизонт ≈ 1/(1−γ) | Коли |
|---|---|---|
| 0 | 1 крок | **короткозорий** агент: тільки миттєва винагорода |
| 0.9 | ~10 кроків | короткі епізоди (gridworld курсу — саме 0.9) |
| 0.99 | ~100 кроків | довгі епізоди, віддалені цілі |
| →1 | ∞ | **далекоглядний**; в неперервних задачах γ<1 обов'язково (збіжність ряду) |

Правило: γ обирається із **довжини причинного ланцюжка** «дія → наслідок».
Якщо винагорода за 50 кроків від дії, γ=0.9 її фактично не бачить (0.9⁵⁰≈0.005).
γ — частина постановки задачі; тюнити його «для збіжності» — маскувати проблему
α або ε.

## Крок 2 — α: крок навчання і його розклад

TD-оновлення: `Q(s,a) ← Q(s,a) + α·(target − Q(s,a))`.

| Симптом | Діагноз | Дія |
|---|---|---|
| Q скаче, крива винагороди пилкою | α завеликий | зменшити (0.5 → 0.1) або увімкнути розклад |
| навчання повзе, плато далеко від оптимуму | α замалий | збільшити старт |
| стохастичне середовище, треба збіжність | α має спадати | `'1/n'` (умови Роббінса-Монро) або `'decay'` |
| нестаціонарне середовище | α сталий | константний α = експоненційне забування — тут це фіча |

Сигнатура нотбука курсу (стартові значення звідти):
`alpha=0.1, alpha_decrease ∈ {None, '1/n', 'decay'}, alpha_decay=0.9, alpha_min=0.1`.

## Крок 3 — ε: розвідка та її згасання

ε-жадібна стратегія: з імовірністю ε — випадкова дія, інакше argmax Q.

| Розклад | Код нотбука | Коли |
|---|---|---|
| константа | `eps=0.1` | базлайн порівнянь курсу; вічна 10%-розвідка |
| `'1/n'` | `eps = 1/i_episode` | швидке згасання; ризик недорозвіданих кутів |
| `'decay'` | `eps *= eps_decay` (0.9–0.95) до `eps_min` (0.2 або 0.01) | **робочий дефолт**: плавно, з підлогою |

Правила: **`eps_min > 0` у стохастичних середовищах** (розвідка не має вмирати);
оцінювати навчену політику — окремим прогоном із `eps≈0.01`, а не тим самим ε,
що вчив. Порівняння курсу SARSA/SARSAmax/Expected SARSA робилось при ε=0.1 —
без вирівняного ε порівняння алгоритмів беззмістовне.

## Крок 4 — вибір алгоритму

| Ситуація | Алгоритм | Чому |
|---|---|---|
| модель середовища p(s',r\|s,a) відома, станів мало | **Policy/Value Iteration** (DP) | точно, без взаємодії |
| моделі немає, епізоди завершуються | **Monte Carlo** | вчиться з повних віддач; без бутстрепу |
| моделі немає, вчитись по кроку | **TD**: SARSA / Q-learning / Expected SARSA | бутстреп по Беллману |
| безпека під час навчання важлива | **SARSA** (on-policy) | вчить політику, ЯКОЮ ходить (з ε) — обережні шляхи |
| потрібна оптимальна жадібна політика | **Q-learning (SARSAmax)** | off-policy: вчить max-політику незалежно від розвідки |
| менша дисперсія за ту саму схему | **Expected SARSA** | матсподівання по π замість вибірки наступної дії |
| простір станів завеликий для таблиці | **DQN** | + два стабілізатори: experience replay і fixed Q-targets |
| неперервні дії / потрібна стохастична політика | **REINFORCE (+baseline) / Actor-Critic** | градієнт політики; baseline ріже дисперсію |

TD-target для довідки: SARSA `R+γQ(s',a')`; SARSAmax `R+γ·max_a Q(s',a)`;
Expected SARSA `R+γ·Σ_a π(a|s')Q(s',a)` (нотбук реалізує всі три однією
функцією `update_Q_sarsa(..., sarsa_method=...)`).

## Режими відмови

- **Порівняння алгоритмів із різними ε/α** — артефакт розкладу, не алгоритму.
- **ε згасло до 0 зарано** — політика замерзла в першому знайденому кутку;
  крива винагороди «плато нижче можливого». Підняти `eps_min`/сповільнити decay.
- **γ тюниться, щоб «стабілізувати»** — маскує завеликий α; спершу α.
- **Оцінка політики з ε навчання** — занижує якість; окремий greedy-прогін.
- **Reward shaping через wrapper** (як RewardWrapper нотбука) — міняє задачу;
  документувати як зміну постановки, не «тюнінг».
- **DQN без replay/fixed targets** — розбіжність через корельовані апдейти;
  це не «ще два трюки», а умови збіжності на практиці.

## Що повідомити

1. Постановка: епізодична/неперервна, горизонт → γ.
2. Стартова трійка з нотбука: `alpha=0.1, gamma=0.9, eps=0.1` +
   `eps_decrease='decay', eps_decay=0.9, eps_min=0.01`, `num_episodes=500`.
3. Алгоритм із таблиці + чому (on/off-policy, безпека, розмір простору).
4. Як міряти: ковзне середнє віддачі за епізод (вікно ~100) на навчанні +
   окремий greedy-прогін для фінальної якості.
