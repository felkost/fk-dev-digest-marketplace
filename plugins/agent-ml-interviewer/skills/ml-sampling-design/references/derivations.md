# Виведення: ваги, design effect, дві школи інференсу

## 1. Вага як 1/p: оцінка Горвіца-Томпсона

Нехай елемент `i` потрапляє у вибірку з відомою ймовірністю `π_i > 0`. Оцінка
суми по популяції:

```
Ŷ_HT = Σ_{i ∈ s} y_i / π_i
```

Незміщеність доводиться через індикатор включення `I_i` з `E[I_i] = π_i`:

```
E[Ŷ_HT] = E[ Σ_i I_i · y_i/π_i ] = Σ_i y_i/π_i · E[I_i] = Σ_i y_i = Y
```

Вага `w_i = 1/π_i` читається як «скільки одиниць популяції представляє цей
рядок». Середнє — відношення `Σ w_i y_i / Σ w_i` (оцінка Гаєка), яке стійкіше
за `Ŷ_HT/N`, коли `N` невідоме точно.

**Умова `π_i > 0` не технічна.** Якщо частина популяції має нульову ймовірність
потрапити (немає у frame), жодна вага її не поверне — це і є межа узагальнення
з Кроку 1 SKILL.md.

### Чому наївне середнє зміщується

Живий прогін (`verify`, n = 200 000): дві страти, значення ≈0 і ≈10, ймовірності
відбору 0.05 і 0.50 відповідно.

| Оцінка | Значення |
|---|---|
| істинне середнє по популяції | 4.9964 |
| наївне середнє по вибірці | **9.0802** |
| зважене `Σwy/Σw` | 4.9720 |

Наївне середнє зсунуте до надвибраної страти. Зміщення тим більше, чим сильніше
різняться `π` **і** чим більша різниця середніх між стратами; якщо страти не
відрізняються за `y`, ваги не змінюють нічого — тому «ваги завжди обов'язкові» —
неправда, обов'язкові вони при **нерівних `π` і залежності `y` від страти**.

## 2. Ваги неповернення: чому це MAR

Фінальна ймовірність потрапити в дані = `π_i · ρ_i`, де `ρ_i` — ймовірність
відповісти. `π_i` відома з дизайну, `ρ_i` — ні, її оцінюють моделлю за
спостереженими `X`:

```
w_i = 1/π_i · 1/ρ̂_i(X_i)
```

Це в точності припущення **MAR**: серед однакових `X` неповернулі не
відрізняються від повернулих за `y`. Якщо люди не відповідають саме через
значення `y` (дохід не називають високодохідні) — це **MNAR**, і жодна вага
за спостереженими `X` не виправляє. Механізми — `ml-missing-data`, Крок 1.

## 3. Design effect: виведення

Модель однофакторної випадкової дії: `y_ij = μ + u_j + e_ij`, де `u_j ~ (0, σ²_b)`
— ефект кластера, `e_ij ~ (0, σ²_w)`. Тоді

```
ICC = σ²_b / (σ²_b + σ²_w)
```

Для `k` кластерів по `m` спостережень середнє `ȳ` має дисперсію

```
Var(ȳ) = (σ²_b + σ²_w/m) / k
```

а наївна формула, що вважає всі `n = km` незалежними, дає `(σ²_b + σ²_w)/(km)`.
Відношення:

```
deff = Var_справжня / Var_наївна = 1 + (m − 1)·ICC
n_eff = n / deff
```

Звідси **SE занижується рівно у `sqrt(deff)` разів** — саме це й перевіряється
чисельно.

### Жива перевірка (statsmodels, `cov_type="cluster"`)

| Кластерів | ICC (оцінена) | deff | SE_клас/SE_наївна | sqrt(deff) | розбіжність |
|---|---|---|---|---|---|
| 40 | 0.2114 | 6.075 | 2.471 | 2.465 | 0.0063 |
| 200 | 0.1782 | 5.278 | 2.298 | 2.297 | 0.0010 |
| 500 | 0.1805 | 5.333 | 2.310 | 2.309 | **0.0004** |

Істинна ICC у всіх трьох — 0.20. Два незалежні шляхи (аналітичний `sqrt(deff)`
і емпіричний кластерний sandwich-SE) сходяться тим точніше, чим більше
кластерів — тобто формула підтверджена, а не підігнана.

**Крайні випадки формули:**

- `ICC = 0` → `deff = 1`: кластери нічого не змінюють, рядки справді незалежні.
- `ICC = 1` → `deff = m`: усередині кластера всі однакові, `n_eff = k`, тобто
  **кількість кластерів і є весь обсяг вибірки**.
- `m = 1` → `deff = 1` за будь-якої ICC: кластер із одного елемента — не кластер.

Практичний наслідок із третього рядка: **збільшувати `m` (більше рядків із тих
самих кластерів) дає спадну віддачу**, а збільшувати `k` (більше кластерів) —
ні. Це прямий орієнтир для планування збору даних.

### Чому мало кластерів псує ще й самі кластерні SE

Sandwich-оцінка коваріації спроможна за `k → ∞`, не за `n → ∞`. При малому `k`
вона зміщена вниз. Виміряно: покриття 95% ДІ на **10 кластерах — 0.937**
(300 повторів). Зміщення помірне, але напрям завжди один — інтервал занадто
впевнений. Конвенція «≥ 40 кластерів» саме звідси; менше — потрібні поправки
(на кшталт Kenward-Roger у змішаних моделях) або обережність у формулюваннях.

## 4. Design-based проти model-based (Sterba 2009)

| | design-based | model-based |
|---|---|---|
| Що випадкове | **включення у вибірку** (`I_i`); `y` фіксовані | **`y`** із моделі; вибірка може бути умовною |
| Популяція | скінченна, конкретна | нескінченна «суперпопуляція» |
| Інструмент | ваги, страти, кластери у формулах SE | предиктори, страти й **взаємодії** в самій моделі |
| Природно оцінює | суми, середні, частки, відношення | будь-які параметри моделі, прогноз |
| Вимога коректності | правильно відомі `π` | правильно **специфікована** модель |

ML майже завжди неявно model-based: усе, що знаємо про дизайн, має ввійти в
модель ознаками. Ключове й найчастіше забуте — **взаємодії**: щоб model-based
відтворив те, що design-based робить вагами, у моделі мають бути не лише
індикатори страт, а і їхні взаємодії з рештою предикторів. Інакше модель
припускає однаковий ефект у всіх стратах, чого зважена оцінка не припускає.

**Практичний гібрид** (дефолт для ML-задач): страти й розмір кластера — ознаки;
кластер — одиниця групування у CV; ваги — у `fit` **і** в scorer; звітний обсяг
— `n_eff`, не `n`.

## 5. Зважування дизайну проти зважування дисбалансу

Один і той самий аргумент `sample_weight` (і `class_weight`) використовують для
двох незалежних цілей:

| Мета | Звідки береться | Що виправляє |
|---|---|---|
| дизайн-вага | 1/π із дизайну вибірки | зміщення оцінки щодо **популяції** |
| вага дисбалансу | 1/частота класу або ручна ціна | асиметрію **ціни помилок** |

Вони не взаємозамінні й **множаться**, якщо потрібні обидві. Плутанина дає
типову відповідь «у нас уже є ваги» на питання про зовсім інші ваги — тому
у звіті вони називаються окремо. Ціна помилок як таких — `ml-metric-choice`
і `ml-decision-threshold`, не цей скіл.
