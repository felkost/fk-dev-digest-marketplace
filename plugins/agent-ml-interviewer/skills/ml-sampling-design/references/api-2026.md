# API: ваги й кластерні похибки — перевірено 2026-07-18

Середовище: **Python 3.12.10 · scikit-learn 1.9.0 · statsmodels 0.14.6**.
Кожен блок нижче — результат живого виклику, не переказ документації.

## 1. Metadata routing: єдиний спосіб зважити метрику

**Дефолт вимкнений:** `sklearn.get_config()["enable_metadata_routing"]` → `False`.

### Що відбувається без routing

```python
cross_validate(est, X, y, cv=cv, params={"sample_weight": w})
```

Ваги **доїжджають до `fit`** (перевірено: коефіцієнт змінюється з +0.011 на
+4.94 на даних, де ваги гасять половину рядків), але **scorer лишається
незваженим** — без ворнінга. На тій самій фікстурі: score 0.5083 без routing
проти **0.9803** із routing і зваженим scorer.

### Робочий рецепт

```python
import sklearn
from sklearn.metrics import get_scorer

with sklearn.config_context(enable_metadata_routing=True):   # або set_config(...)
    est = LogisticRegression().set_fit_request(sample_weight=True)
    sc  = get_scorer("accuracy").set_score_request(sample_weight=True)
    res = cross_validate(est, X, y, cv=cv,
                         params={"sample_weight": w}, scoring=sc)
```

`GridSearchCV` за тим самим правилом — перевірено:

```python
gs = GridSearchCV(est, {"C": [0.1, 1]}, cv=3, scoring=sc).fit(X, y, sample_weight=w)
```

### Пастка: увімкнення routing ЛАМАЄ робочий Pipeline

З `enable_metadata_routing=True` виклик `pipe.fit(X, y, sample_weight=w)` кидає
**`UnsetMetadataPassedError`**: кожен крок мусить явно заявити свою позицію,
включно з тими, кому ваги не потрібні.

```python
# ПРАВИЛЬНО: кожен крок заявляє намір
pipe = Pipeline([
    ("s", StandardScaler().set_fit_request(sample_weight=False)),   # явна відмова
    ("m", LogisticRegression().set_fit_request(sample_weight=True)),
])
pipe.fit(X, y, sample_weight=w)      # працює
```

**Без** routing той самий Pipeline вимагає префікс кроку, а голий аргумент
кидає `ValueError`:

```python
pipe.fit(X, y, m__sample_weight=w)   # єдиний робочий варіант без routing
pipe.fit(X, y, sample_weight=w)      # ValueError: Pipeline.fit does not accept ...
```

Тобто синтаксис **різний у двох режимах** — код, скопійований із прикладу під
інший режим, або впаде, або (гірше) тихо не зважить метрику.

## 2. Хто приймає `sample_weight` у `fit`

| Клас | `sample_weight` |
|---|---|
| `LogisticRegression` | приймає |
| `RandomForestClassifier` | приймає |
| `HistGradientBoostingClassifier` | приймає |
| `SVC` | приймає |
| **`KNeighborsClassifier`** | **`TypeError: fit() got an unexpected keyword argument 'sample_weight'`** |

kNN зважує сусідів (`weights="distance"`), але **ваги спостережень не підтримує
взагалі**. Якщо дизайн-ваги суттєві, це аргумент проти сімейства kNN, а не
привід ігнорувати ваги (`ml-model-selection`).

## 3. Кластерні (робастні) стандартні похибки — statsmodels

```python
import statsmodels.api as sm
fit = sm.OLS(y, sm.add_constant(X)).fit(
    cov_type="cluster", cov_kwds={"groups": cluster_id})
```

- `groups` — масив довжини `n` з ідентифікатором кластера.
- Точкові оцінки **не змінюються** — змінюються лише SE, ДІ й p-values.
- Перевірено: SE середнього 0.0701 (кластерна) проти 0.0317 (наївна),
  відношення 2.216 при `sqrt(deff) = 2.211`.
- `add_constant` обов'язковий — той самий урок, що й з VIF
  (`ml-linear-regularization`).

Зважена регресія — окремий клас: `sm.WLS(y, X, weights=w)`. Ваги WLS
(обернені до дисперсії) й ваги дизайну (обернені до `π`) збігаються за формою,
але не за змістом; для дизайн-ваг коректні SE все одно вимагають
`cov_type="cluster"` або реплікаційних методів.

## 4. Дрібниці, які економлять годину

- `train_test_split(..., stratify=y)` стратифікує **спліт**, а не відбір; до
  дизайну вибірки стосунку не має — це просто збереження часток класів.
- `StratifiedKFold` — те саме: страта тут = клас цілі, не страта дизайну. Якщо
  страти дизайну не збігаються з класами, потрібне окреме групування.
- `class_weight="balanced"` рахується **з навчальних даних** і не знає нічого
  про ймовірності відбору; складати його з дизайн-вагами вручну (множенням)
  доводиться самому.
- Для оцінок «на популяцію» після моделювання (post-stratification, raking)
  у sklearn немає нічого; це `statsmodels`/`samplics`/ручний перерахунок.
