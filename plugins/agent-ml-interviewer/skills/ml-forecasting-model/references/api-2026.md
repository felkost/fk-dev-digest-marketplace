# API: forecasting — перевірено 2026-07-18

Середовище: **Python 3.12.10 · statsmodels 0.14.6 · scikit-learn 1.9.0 ·
lightgbm 4.6.0**. Усе нижче — живі виклики.

## 1. Стаціонарність

```python
from statsmodels.tsa.stattools import adfuller, kpss
adf_stat, adf_p, *_ = adfuller(y, autolag="AIC")     # H0: одиничний корінь
kpss_stat, kpss_p, *_ = kpss(y, regression="c", nlags="auto")  # H0: стаціонарний
```

- **`kpss` видає `InterpolationWarning`**, коли p за межами таблиці
  (`"p-value is greater than the indicated value"`): це не помилка, p насичена
  на 0.10 (згори) або 0.01 (знизу). Ловити: `warnings.catch_warnings()`.
- `regression="c"` — стаціонарність навколо константи; `"ct"` — навколо тренду
  (для перевірки trend-stationary).
- ADF `autolag="AIC"` сам обирає число лагів; фіксувати `maxlag` лише навмисне.

## 2. Класичні моделі

```python
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.tsa.statespace.sarimax import SARIMAX
from statsmodels.tsa.holtwinters import ExponentialSmoothing

# ARIMA(p,d,q)
res = ARIMA(y_train, order=(2, 1, 2)).fit()
fc  = res.forecast(steps=H)                      # точковий прогноз
ci  = res.get_forecast(H).conf_int()             # інтервал прогнозу

# SARIMAX із сезоном і екзогенними X
res = SARIMAX(y_train, exog=X_train,
              order=(1,1,1), seasonal_order=(1,1,1,12)).fit(disp=False)
fc  = res.forecast(H, exog=X_future)             # X_future ОБОВ'ЯЗКОВИЙ

# ETS / Holt-Winters
res = ExponentialSmoothing(y_train, trend="add",
        seasonal="add", seasonal_periods=12).fit()
fc  = res.forecast(H)
```

- **`SARIMAX.forecast(exog=...)` вимагає майбутні `X`** — якщо їх нема в проді,
  екзогенні предиктори непридатні (їх самих треба прогнозувати).
- `ExponentialSmoothing`: `trend`/`seasonal` ∈ `{"add","mul",None}`;
  `"mul"` тільки для строго додатних рядів (інакше `ValueError`).
- `.fit(disp=False)` глушить лог оптимізатора SARIMAX.
- Довгі ряди + високий порядок → `ConvergenceWarning`; це сигнал спростити
  порядок, не ігнорувати.

## 3. Global ML: лаги + gradient boosting

`pmdarima`, `prophet`, `sktime`, `darts`, `neuralprophet` — **у середовищі
відсутні** (перевірено імпортом). Global-forecasting будується руками на
`lightgbm` (є, 4.6.0) або `sklearn`:

```python
import numpy as np
from lightgbm import LGBMRegressor

def make_lags(series, L=12):
    X = np.column_stack([series[i:len(series)-L+i] for i in range(L)])
    return X, series[L:]

# global: складаємо лаг-ознаки з БАГАТЬОХ рядів у одну навчальну матрицю
Xs, ys = [], []
for s in all_series:
    X, y = make_lags(s)
    Xs.append(X); ys.append(y)
model = LGBMRegressor(n_estimators=100, verbose=-1).fit(np.vstack(Xs),
                                                        np.concatenate(ys))

# рекурсивний прогноз на H кроків для одного ряду
hist = list(one_series)
for _ in range(H):
    x = np.array(hist[-12:]).reshape(1, -1)
    hist.append(model.predict(x)[0])
preds = hist[-H:]
```

- `LGBMRegressor(verbose=-1)` — глушить епохний лог.
- **Ідентичність ряду подавати ознакою** (статичні атрибути, one-hot/ID),
  інакше global-модель зливає всі ряди в один.
- Лаг-ознаки будуються ТІЛЬКИ з минулого — інакше витік (`ml-validation-design`).

**Втрати для переміжного/скошеного попиту (перевірено живим викликом, 4.6.0):**

```python
LGBMRegressor(objective="tweedie", tweedie_variance_power=1.5)  # 1<p<2
LGBMRegressor(objective="poisson")                              # лічильні
LGBMRegressor(objective="quantile", alpha=0.9)                  # квантиль (інтервали)
```

- `tweedie_variance_power` у (1, 2); дефолт 1.5. `p→1` ближче до Poisson,
  `p→2` до Gamma. Прогноз клемпити `np.clip(pred, 0, None)` — від'ємних продажів
  не буває.
- `objective="tweedie"/"poisson"/"quantile"` **усі доступні** в 4.6.0 (на відміну
  від деяких sklearn-об'єктів). `sklearn.linear_model.TweedieRegressor(power=,
  link="log")` теж є — для лінійної альтернативи.
- Для інтервалів прогнозу без statsmodels — три `quantile`-моделі (0.1/0.5/0.9).
- Тюнінг самого бустингу — `ml-tree-ensemble-params` (n_estimators↔learning_rate).

## 4. Метрики

```python
from sklearn.metrics import mean_absolute_error, mean_squared_error
# MASE вручну — у sklearn немає
def mase(y_true, y_pred, y_train, season=1):
    naive = np.mean(np.abs(y_train[season:] - y_train[:-season]))
    return mean_absolute_error(y_true, y_pred) / naive
```

- **`mean_absolute_percentage_error` є в sklearn, але вибухає на нулях**
  (перевірено: 1.1·10¹⁵ з одним нулем у `y_true`). При нулях/від'ємних — MASE.
- **`mean_squared_error(..., squared=False)` у 1.9 кидає `TypeError`** —
  параметр `squared` **видалено** (не deprecated з ворнінгом, а прибрано;
  перевірено живцем). RMSE тепер лише через `root_mean_squared_error` (є в 1.9).
  Той самий клас зламу, що й `AdaBoost(algorithm=...)`.
- MASE в sklearn немає — рахувати вручну (код вище); `season` = період для
  сезонного наївного знаменника.

## 5. Валідація (делеговано, але тут — сплітер)

Готового rolling-origin для forecasting у sklearn — `TimeSeriesSplit`, але він
без сезонного вирівнювання. Схема, gap, purge — `ml-validation-design`, Кроки
3/3a. Тут лише правило: **горизонт test = горизонту прода**, і тестувати
рекурсивно, а не на H=1.
