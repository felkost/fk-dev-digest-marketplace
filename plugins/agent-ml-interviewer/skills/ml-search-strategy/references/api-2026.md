## sklearn: чотири пошукачі

```python
from sklearn.model_selection import GridSearchCV, RandomizedSearchCV
# Halving і в 1.9 ЛИШАЄТЬСЯ experimental — БЕЗ цього імпорту ImportError
# (перевірено: статус не змінився між 1.6 і 1.9, попри дозрівання решти 1.9-фіч):
from sklearn.experimental import enable_halving_search_cv  # noqa
from sklearn.model_selection import HalvingGridSearchCV, HalvingRandomSearchCV
```

```python
from sklearn.pipeline import Pipeline
from sklearn.model_selection import StratifiedKFold
from scipy.stats import loguniform

pipe = Pipeline([("prep", preprocessor), ("clf", model)])   # префікс "clf__" у сітці
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=1)

GridSearchCV(pipe, {"clf__max_depth": range(1, 8)}, cv=skf,
             scoring="roc_auc", n_jobs=-1, return_train_score=True)

RandomizedSearchCV(pipe, {"clf__C": loguniform(1e-3, 30)}, n_iter=100,
                   cv=skf, scoring="roc_auc", random_state=1, n_jobs=-1)

HalvingGridSearchCV(pipe, grid, factor=3, min_resources="exhaust", cv=skf)
```

Після фіту: `best_params_`, `best_score_` (CV, зміщений угору), `best_estimator_`
(перефічений на всьому train через `refit=True`), `cv_results_`.

**Callbacks (нове в 1.9) — реальна форма API, перевірена живцем.** Це не
конструкторний параметр і не `set_params(callbacks=...)` (обидва варіанти
падають з помилкою) — а окремий метод `set_callbacks(*callbacks)`, що приймає
callback-и як позиційні аргументи, не список:

```python
from sklearn.callback import ProgressBar

search = GridSearchCV(pipe, grid, cv=skf, scoring="roc_auc")
search.set_callbacks(ProgressBar())      # *args, НЕ ProgressBar([...])
search.fit(X, y)
```

Працює однаково на `GridSearchCV`, `RandomizedSearchCV`, `HalvingGridSearchCV`,
`HalvingRandomSearchCV` (усі успадковують `CallbackSupportMixin`) — перевірено
на всіх чотирьох. `ScoringMonitor` — той самий виклик, для логування скору
по ітераціях замість прогрес-бару.

`LogisticRegressionCV.scoring` дефолт **уже видає `FutureWarning` у 1.9**
(дослівно: *«The default value of the parameter 'scoring' will change from
None, i.e. accuracy, to 'neg_log_loss' in version 1.11»*) — задавайте `scoring`
явно вже зараз, інакше кожен фіт друкує попередження.

## Спліттери

```python
from sklearn.model_selection import (KFold, StratifiedKFold, GroupKFold,
                                     TimeSeriesSplit, RepeatedStratifiedKFold)
GroupKFold(5).split(X, y, groups=user_id)     # групи НЕ перетинають фолди
TimeSeriesSplit(5, gap=0)                     # train завжди раніше за val
```

**`GroupKFold` у 1.9 отримав `shuffle`/`random_state`** (перевірено: у 1.6 їх не
було, тепер `GroupKFold(n_splits=5, shuffle=True, random_state=1)` працює й
дає відтворюваний розподіл груп по фолдах). Раніше для перемішаних груп треба
було брати `GroupShuffleSplit`; тепер це не обов'язково, якщо влаштовує саме
K-fold-семантика. Вкладена CV:

```python
from sklearn.model_selection import cross_val_score
inner = StratifiedKFold(5, shuffle=True, random_state=1)
outer = StratifiedKFold(5, shuffle=True, random_state=2)
search = GridSearchCV(pipe, grid, cv=inner, scoring="roc_auc")
honest = cross_val_score(search, X, y, cv=outer, scoring="roc_auc")  # для звіту
```

## Optuna 4.9

### Що deprecated (перевірено живим ворнінгом)

`TPESampler(gamma=...)` →
`'gamma' has been deprecated in v4.9.0. This feature will be removed in v6.0.0`.
Так само: `prior_weight`, `consider_magic_clip`, `consider_endpoints`, `weights`,
`warn_independent_sampling`; `categorical_distance_func` мігрує в OptunaHub.
**Не пишіть їх у нового коду** — це найчастіша галюцинація моделей, навчених
на до-2026 матеріалах.

### Сучасний шаблон

```python
import optuna
from optuna.samplers import TPESampler
from optuna.pruners import MedianPruner
from sklearn.model_selection import cross_val_score

def objective(trial: optuna.Trial) -> float:
    params = {
        "max_depth":     trial.suggest_int("max_depth", 2, 12),
        "learning_rate": trial.suggest_float("learning_rate", 1e-3, 0.3, log=True),
        "max_features":  trial.suggest_float("max_features", 0.3, 0.9),
    }
    model = make_model(**params)                 # усередині Pipeline!
    score = cross_val_score(model, X, y, cv=skf, scoring="roc_auc", n_jobs=-1)
    return score.mean()

study = optuna.create_study(
    direction="maximize",
    sampler=TPESampler(multivariate=True, group=True, seed=1),
    pruner=MedianPruner(n_warmup_steps=5),
    storage="sqlite:///study.db", study_name="rf_adult", load_if_exists=True,
)
study.optimize(objective, n_trials=100, timeout=1800)
study.best_params, study.best_value
```

Прунінг усередині ітеративного навчання: `trial.report(score_t, step=t)` +
`if trial.should_prune(): raise optuna.TrialPruned()`.

Умовні простори — сила TPE проти сітки:

```python
kind = trial.suggest_categorical("model", ["rf", "logreg"])
if kind == "logreg":
    trial.suggest_float("C", 1e-3, 30, log=True)
```

## Бюджет у фітах

`фітів = кандидатів × n_splits (+ refit)`. Сітка 6×5×4 з 5-fold = 600 фітів.
Random/Optuna із 100 trials × 5 фолдів = 500. Halving економить: більшість
кандидатів бачить лише малий ресурс.

## Пастки

| Пастка | Захист |
|---|---|
| selector/scaler поза Pipeline | усе, що фітиться, — у Pipeline під пошуком |
| `HalvingGridSearchCV` ImportError | experimental-імпорт (рядок вище) — і в 1.9 теж |
| `GridSearchCV(callbacks=[...])` або `.set_params(callbacks=...)` | `search.set_callbacks(cb1, cb2)` — метод, *args |
| `TPESampler(gamma=..., weights=...)` | deprecated 4.9 → сучасний шаблон |
| подвійний `n_jobs=-1` (пошук × модель) | паралельте пошук, модель `n_jobs=None` |
| лінійна сітка по C/alpha/lr | `np.logspace` / `loguniform` / `log=True` |
| `best_score_` у звіт | test або вкладена CV |
| random search без `random_state` / TPE без `seed` | фіксуйте |
