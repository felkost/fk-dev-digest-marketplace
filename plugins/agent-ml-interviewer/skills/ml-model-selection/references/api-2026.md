## HistGradientBoosting: категорії й пропуски нативно

`HistGradientBoostingClassifier` має параметр **`categorical_features`**
(перевірено) і сам обробляє `NaN` — тому для змішаних табличних даних це
найкоротший шлях від сирої таблиці до моделі:

```python
HistGradientBoostingClassifier(
    categorical_features="from_dtype",   # або список індексів/імен
    max_iter=300, learning_rate=0.05, early_stopping=True,
    random_state=0,
)
```

Решті деревних моделей sklearn категорії треба кодувати самому.

## Що вимагає скейлінгу (і чому це в Pipeline)

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

Pipeline([("sc", StandardScaler()), ("clf", KNeighborsClassifier(n_neighbors=15))])
Pipeline([("sc", StandardScaler()), ("clf", SVC(kernel="rbf", C=1.0))])
Pipeline([("sc", StandardScaler()), ("clf", LogisticRegression(max_iter=1000))])
```

kNN і SVM працюють на відстанях; лінійні з регуляризацією — бо штраф залежить від
шкали. Дерева й бустинг скейлінгу не потребують. Scaler поза Pipeline і зафічений
до спліту — витік (див. `ml-validation-design`).

## Ймовірності там, де їх немає напряму

```python
from sklearn.calibration import CalibratedClassifierCV
CalibratedClassifierCV(LinearSVC(), method="sigmoid", ensemble=False)
```

Для SVC не вмикати `probability=True` — параметр **deprecated у 1.9**
(див. `ml-decision-threshold/references/api-2026.md`).

## Інкрементальне навчання, коли дані не влазять

```python
clf = SGDClassifier(loss="log_loss")     # логістична через SGD
for X_chunk, y_chunk in chunks:
    clf.partial_fit(X_chunk, y_chunk, classes=[0, 1])
```

`HistGradientBoostingClassifier` теж придатний для великих обсягів (гістограмний
алгоритм), але цілком у пам'яті.

## Чесне порівняння кандидатів

```python
from sklearn.model_selection import cross_validate
for name, pipe in candidates.items():
    r = cross_validate(pipe, X, y, cv=cv, scoring="average_precision")
    print(name, r["test_score"].mean().round(4), "±", r["test_score"].std().round(4))
```

Один `cv`-об'єкт на всіх кандидатів; різниця, менша за σ по фолдах, не є
різницею. Тюнінг — після відбору сімейства, інакше порівнюються бюджети пошуку.
