# Скільки кластерів і які параметри кластеризації

Без міток «правильного k» не існує — є **збіг кількох незалежних сигналів**
плюс огляд очима. Скіл організовує саме це.

**Відповіді українською; ідентифікатори й код — англійською.**

## Не відповідає на

- **Зниження розмірності до кластеризації** → `ml-dimensionality-features`.
- **Оцінка supervised-моделі** → `ml-metric-choice`.

## Крок 0 — передумови, без яких числа брехатимуть

1. **Масштабування обов'язкове**: k-means та відстані живуть у евклідовій
   метриці; ознака в гривнях задавить ознаку в частках. `StandardScaler` — дефолт.
2. **Форма кластерів = вибір алгоритму**: k-means шукає опуклі сфери однакового
   радіуса. Кільця/півмісяці/різні щільності — це DBSCAN/OPTICS/GMM/spectral,
   і жоден вибір k цього не полагодить.
3. Категорії у k-means не живуть (середнє від one-hot — не центр) — k-modes або
   інша постановка.

## Крок 1 — вибір k: три сигнали разом

```
python ml-clustering-k/scripts/cluster_k_report.py --csv d.csv --k-min 2 --k-max 12
python ml-clustering-k/scripts/cluster_k_report.py --self-test
```

| Сигнал | Що міряє | Читання |
|---|---|---|
| **Elbow / inertia (SSE)** | компактність | коліно через `KneeLocator(curve="convex", direction="decreasing")` |
| **Silhouette** | відділеність (b−a)/max(a,b) ∈ [−1,1] | максимум по k; **старт із k=2** — для k=1 не визначений |
| **Davies-Bouldin** | відношення внутр./міжкластерної | мінімум по k |

**Розбіжність сигналів — інформація, не поломка**: elbow каже 3, silhouette 2 —
ймовірно, два великі кластери з підструктурою; дивитись silhouette-ножі по
кластерах і вирішувати на рівні задачі. Кластерів більше, ніж класів, — валідний
результат (внутрішньокласові режими).

**Урок «106 хвилин»**: у нотбуці курсу повний silhouette-свіп по MNIST зайняв
~106 хв. Silhouette — O(n²): скрипт рахує його на **підвибірці** (кап 5000 за
замовчуванням), inertia — на всіх даних. Для n > ~50k беріть `--minibatch`.

## Крок 2 — параметри k-means

| Ручка | Дефолт/старт | Навіщо |
|---|---|---|
| `init` | **`"k-means++"`** | лікує головну слабкість випадкової ініціалізації |
| `n_init` | `"auto"` (1.6: 10 для random, 1 для k-means++) | перезапуски проти локальних мінімумів |
| `max_iter` / `tol` | 300 / 1e-4 | майже ніколи не чіпати |
| `random_state` | фіксувати | центри відтворювані |
| `MiniBatchKMeans(batch_size=4096)` | великі n | нотбук курсу: MNIST лише так |

## Крок 3 — коли k-means не той інструмент

| Симптом даних | Алгоритм | Ключові параметри (дефолти сітки нотбука) |
|---|---|---|
| еліпси/різні розміри, потрібні ймовірності | **GaussianMixture** | `n_components` (той самий свіп), `covariance_type="full"` |
| довільна форма + шум/викиди | **DBSCAN** | `eps` (з k-distance графіка), `min_samples≈2·d`; нотбук: eps .15–.30 |
| щільності різні | **OPTICS** | `min_samples` 5–20, `xi` .035–.25, `min_cluster_size` .1–.2 |
| потрібна ієрархія/дендрограма | **Agglomerative** | `linkage="ward"` (евклід); `connectivity=kneighbors_graph(...)` для локальності |
| графова структура | Spectral | `eigen_solver="arpack"` |

GMM додатково: вибір `n_components` за **BIC/AIC** — законна альтернатива
elbow, бо є правдоподібність.

## Крок 3a — що ви насправді питаєте, кластеризуючи людей (S6E04)

Кластеризація — це не техніка, а **питання про однорідність**: чи взаємозамінні
суб'єкти, чи є підгрупи, для яких «середнє по всіх» не тримається. Це кінець
континууму, не дихотомія «person-centered проти variable-centered»:

| Рівень | Що припускає | Метод |
|---|---|---|
| variable-centered | один β/центр на всіх (взаємозамінність) | GLM, одна регресія |
| **класифікація** | підгрупи, всередині однорідні | **k-means, GMM, LCA/LPA** |
| person-centered | кожен унікальний | dynamic/p-technique factor |

- **LCA/LPA — це model-based двійник k-means** (латентні класи/профілі): ті самі
  підгрупи, але через правдоподібність, тож `n_classes` обирається за BIC, а не
  силуетом. GMM — окремий випадок LPA з нормальними компонентами.
- **Пастка інтерпретації:** знайшовши кластери, їх майже завжди описують
  **середніми ознак** (histogram/ANOVA по класах) — тобто повертаються до
  variable-centered мислення. Це нормально, але усвідомлено: назвати кластер
  «тривожні відторгнуті» — уже про змінні, не про людей.
- **Ергодичність — прихована умова.** «Один β на всіх» законний лише якщо процес
  однорідний і стаціонарний (ергодичний). У соцданих це майже ніколи не так — і
  саме тому підгрупи або **параметрична модерація** (β як функція ознак,
  високовимірна взаємодія) часто чесніші за єдиний центр.

## Крок 4 — перевірка результату

- **Silhouette-ножі по кластерах**: кластер із ножем нижче середнього і
  «від'ємним хвостом» — кандидат на злиття.
- **Стабільність**: інший `random_state`/підвибірка → ті самі кластери? Ні —
  k завищене або алгоритм не той.
- **Очі**: 2D-проєкція (PCA) з розфарбуванням — обов'язкова; проєкція —
  гіпотеза, не доказ.
- Мітки к-means залежать від перестановки — порівнювати розбиття через ARI/AMI,
  не порівнянням номерів.

## Режими відмови

- **Свіп без масштабування** — k підлаштовується під найширшу ознаку.
- **Silhouette від k=1** — не визначений; скрипт стартує з 2.
- **Elbow на увігнутій кривій** — коліна може не бути; KneeLocator поверне None,
  це чесна відповідь «сигнал не голосує».
- **DBSCAN eps підібраний на нескейлених даних** — переносити не можна.
- **Повний silhouette на 100k+** — години; підвибірка або Davies-Bouldin.
- **«k = кількість класів бізнесу»** — кластери відповідають структурі даних,
  а не оргструктурі; мапінг — окремий крок після.

## Що повідомити

1. Таблицю трьох сигналів по k і **чи вони згодні**.
2. Рекомендоване k + друге-найкраще з причиною.
3. Параметри запуску (init, n_init, scaler) для відтворення.
4. Якщо форма не сферична — який алгоритм замість k-means і його стартові.
