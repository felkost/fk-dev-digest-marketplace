Вручну — кістяк той самий:

```python
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans, MiniBatchKMeans
from sklearn.metrics import silhouette_score, silhouette_samples, davies_bouldin_score
from kneed import KneeLocator

Xs = StandardScaler().fit_transform(X)               # ОБОВ'ЯЗКОВО до відстаней
sse = {}
for k in range(2, 13):                               # silhouette: старт із 2
    km = KMeans(n_clusters=k, init="k-means++", n_init=10, random_state=1).fit(Xs)
    sse[k] = km.inertia_
elbow = KneeLocator(list(sse), list(sse.values()),
                    curve="convex", direction="decreasing").elbow   # може бути None
```

`n_init` у 1.6: `"auto"` = 10 для `init="random"`, 1 для `"k-means++"`; для
відтворюваності зі старими прикладами ставте число явно. Великі n:
`MiniBatchKMeans(batch_size=4096)` (нотбук: MNIST лише так; повний silhouette
там коштував ~106 хв — рахуйте його на підвибірці).

sv = silhouette_samples(Xs, labels)

## GMM: вибір компонент за BIC

```python
from sklearn.mixture import GaussianMixture
bic = {k: GaussianMixture(n_components=k, covariance_type="full",
                          n_init=3, random_state=1).fit(Xs).bic(Xs)
       for k in range(2, 13)}
k_bic = min(bic, key=bic.get)
```

`covariance_type`: `"full"` — еліпси довільної орієнтації (дефолт вибору),
`"diag"` — осі-паралельні (менше параметрів), `"tied"`/`"spherical"` — економія
для малих n. `predict_proba` — м'які належності, чого k-means не дає.

## DBSCAN / OPTICS

```python
from sklearn.cluster import DBSCAN, OPTICS
from sklearn.neighbors import NearestNeighbors

# eps із k-distance графіка: відстань до min_samples-го сусіда, відсортована
d, _ = NearestNeighbors(n_neighbors=min_samples).fit(Xs).kneighbors(Xs)
kdist = np.sort(d[:, -1])                 # коліно цієї кривої ~ eps
DBSCAN(eps=eps, min_samples=2 * n_features).fit(Xs)   # label -1 = шум, це фіча

OPTICS(min_samples=20, xi=0.05, min_cluster_size=0.1).fit(Xs)
```

Сітка дефолтів порівняльного нотбука курсу (стартові точки, не істина):
`{'quantile': .3, 'eps': .3, 'damping': .9, 'preference': -200,
'n_neighbors': 10, 'n_clusters': 3, 'min_samples': 20, 'xi': 0.05,
'min_cluster_size': 0.1}`; по датасетах eps .15–.18, xi .035–.25.

## Ієрархічна

```python
from sklearn.cluster import AgglomerativeClustering
from sklearn.neighbors import kneighbors_graph

conn = kneighbors_graph(Xs, n_neighbors=10, include_self=False)
AgglomerativeClustering(n_clusters=k, linkage="ward", connectivity=conn).fit(Xs)
# дендрограма: scipy.cluster.hierarchy.linkage(Xs, method="ward") + dendrogram(...)
```

`ward` — лише евклід; `average`/`complete` — будь-яка метрика
(`metric="cityblock"` у нотбуці). Теорія злиттів — рекурентність
Lance-Williams (Л18): α_i, α_j, β, γ породжують single/complete/average/
centroid/Ward як окремі випадки.

## Пастки

| Пастка | Захист |
|---|---|
| відстані без скейлінгу | StandardScaler першим кроком, завжди |
| silhouette від k=1 | старт із k=2 |
| повний silhouette на n≥50k | підвибірка (`--silhouette-cap`) або DB |
| elbow «мусить» існувати | KneeLocator поверне None — сигнал не голосує |
| порівняння міток номерами | ARI/AMI |
| eps перенесений між датасетами | k-distance графік на кожному |
| k-means на кільцях/півмісяцях | DBSCAN/OPTICS/spectral, не тюнінг k |
| категорії в k-means | k-modes або інша постановка |
