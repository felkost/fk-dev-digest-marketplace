# API: пропущені дані та імпутація

**Перевірено 2026-07-18 живими викликами на scikit-learn 1.9.0** — сигнатури
зчитані з `inspect.signature`, поведінка відтворена на мінімальних масивах.
Жоден факт нижче не взято з документації чи release notes.

## Сигнатури (1.9.0, дослівно)

| Клас | Параметри |
|---|---|
| `SimpleImputer` | `missing_values, strategy, fill_value, copy, add_indicator, keep_empty_features` |
| `KNNImputer` | `missing_values, n_neighbors, weights, metric, copy, add_indicator, keep_empty_features` |
| `MissingIndicator` | `missing_values, features, sparse, error_on_new` |
| `IterativeImputer` | `estimator, missing_values, sample_posterior, max_iter, tol, n_nearest_features, initial_strategy, fill_value, imputation_order, skip_complete, min_value, max_value, verbose, random_state, add_indicator, keep_empty_features` |

**Жодного `DeprecationWarning`/`FutureWarning` на імпутерах у 1.9** — перевірено
`warnings.catch_warnings(record=True)` на `SimpleImputer` і `KNNImputer`. На тлі
решти пака (де в 1.9 багато застарілостей) імпутери стабільні.

## `IterativeImputer` досі experimental

```python
from sklearn.impute import IterativeImputer
# ImportError: IterativeImputer is experimental and the API might change
# without any deprecation cycle. To use it, you need to explicitly import
# enable_iterative_imputer
```

Робочий імпорт:

```python
from sklearn.experimental import enable_iterative_imputer  # noqa: F401
from sklearn.impute import IterativeImputer
```

Той самий статус, що й у `HalvingGridSearchCV` (`ml-search-strategy`) — між 1.6
і 1.9 не змінився. `SimpleImputer` і `KNNImputer` experimental-імпорту **не**
потребують.

## Пастка 1: `None` в object-колонці не заповнюється

Найнебезпечніша з знайдених, бо тиха і суперечить тому, що показує EDA.

```python
df = pd.DataFrame({"cat": ["a", None, "b", "a"]})
df["cat"].isna().sum()          # -> 1   (pandas ВВАЖАЄ це пропуском)
SimpleImputer(strategy="most_frequent").fit_transform(df)
# -> array([['a'], [None], ['b'], ['a']], dtype=object)
#           ^^^^^^^^^^^ None ЛИШИВСЯ і піде в модель
```

Причина: дефолт `missing_values=np.nan`, а в комірці лежить фізично
`NoneType`, не `float('nan')` — збігу немає. Обидва лікують:

```python
SimpleImputer(missing_values=None, strategy="most_frequent")   # -> ['a','a','b','a']
df["cat"] = df["cat"].fillna(np.nan)                            # привести до nan
```

Контроль: якщо в тій самій колонці лежить `np.nan`, дефолтний імпутер
заповнює нормально (`['a','a','b','a']`) — тобто розбіжність саме між `None` і
`np.nan`, а не між типами колонок.

## Пастка 2: колонка з суцільних NaN зникає мовчки

```python
X = np.array([[1.0, np.nan], [2.0, np.nan]])       # друга колонка вся NaN
SimpleImputer().fit_transform(X).shape[1]           # -> 1  (було 2!)
# UserWarning: Skipping features without any observed values: [1]
```

Форма виходу змінюється — усе, що далі спирається на кількість колонок
(індекси, імена, `ColumnTransformer`), ламається неочікувано. Лік:

```python
SimpleImputer(keep_empty_features=True).fit_transform(X)
# -> колонок 2, порожня заповнена нулями: [1., 0.]
```

## Пастка 3: `add_indicator` бачить лише train

Індикатори створюються **тільки** для колонок, що мали NaN на `fit`.

```python
Xtr = np.array([[1., 2.], [3., 4.], [5., 6.]])      # NaN немає
Xte = np.array([[np.nan, 2.], [3., np.nan]])        # NaN є

si = SimpleImputer(add_indicator=True).fit(Xtr)
si.indicator_.features_        # -> array([], dtype=int64)  — жодного індикатора
si.transform(Xte).shape        # -> (2, 2), а не (2, 4)
si.transform(Xte)              # -> [[3., 2.], [3., 4.]]  NaN заповнено тихо
```

Виняток не кидається, попередження немає. Якщо пропуски в проді з'являються в
колонках, чистих на навчальному зрізі, сигнал «тут був пропуск» буде втрачено
саме там, де він найцінніший. `MissingIndicator(error_on_new=True)` — окремий
шлях отримати помилку замість тиші.

**Нормальна поведінка для порівняння** (NaN є в train):

```python
X = np.array([[1., 2.], [np.nan, 3.], [7., 6.], [4., np.nan]])
SimpleImputer(add_indicator=True).fit_transform(X).shape   # -> (4, 4): 2 + 2
```

## `set_output(transform="pandas")`: імена індикаторів

```python
si = SimpleImputer(add_indicator=True).set_output(transform="pandas")
si.fit_transform(pd.DataFrame({"a": [1., np.nan, 3.], "b": [np.nan, 2., 3.]}))
```

```
     a    b  missingindicator_a  missingindicator_b
0  1.0  2.5                 0.0                 1.0
1  2.0  2.0                 1.0                 0.0
2  3.0  3.0                 0.0                 0.0
```

Імена індикаторів — `missingindicator_<оригінальна_колонка>`. Без
`set_output` повертається безіменний numpy-масив, і відповідність індикаторів
колонкам треба відновлювати через `indicator_.features_`.

## `sample_posterior`: основа множинної імпутації

Одна імпутація приховує невизначеність; `sample_posterior=True` тягне значення
з апостеріорного розподілу, тому різні `random_state` дають різні заповнення.

```python
ii = IterativeImputer(sample_posterior=True, random_state=SEED, max_iter=10)
```

Виміряно на тій самій комірці `X[1, 0]` (n=5, дві колонки):

| `random_state` | `sample_posterior=True` | `sample_posterior=False` |
|---|---|---|
| 0 | 2.552 | 2.391 |
| 1 | 2.104 | 2.391 |
| 2 | 2.662 | 2.391 |

Розкид ~0.56 між прогонами — це і є кількісна ціна пропуску. З
`sample_posterior=False` значення детерміноване й ця інформація втрачається.

## Нативна підтримка NaN у моделях

Перевірено `fit()` на `X` з NaN, без жодної імпутації:

| Модель | Результат |
|---|---|
| `HistGradientBoostingClassifier` | **навчається** |
| `DecisionTreeClassifier` | **навчається** |
| `RandomForestClassifier` | **навчається** |
| `GradientBoostingClassifier` | `ValueError: Input X contains NaN` |
| `LogisticRegression` | `ValueError: Input X contains NaN` |

Найнеочевидніше: **`RandomForest`/`DecisionTree` NaN приймають, а старий
`GradientBoosting` — ні**, хоча обидва деревні. Тобто «деревна модель
впорається з пропусками» — хибне узагальнення; залежить від конкретного класу.
Перевіряйте свій, а не сімейство загалом.

## Обов'язково в Pipeline

Імпутер має параметри, що вчаться з даних (середнє, моделі `IterativeImputer`,
сусіди `KNNImputer`), тому `fit` поза CV — витік. Канон:

```python
Pipeline([
    ("imp", SimpleImputer(strategy="median", add_indicator=True)),
    ("sc", StandardScaler()),
    ("clf", LogisticRegression(max_iter=1000)),
])
```

Стратегію імпутації можна тюнити як гіперпараметр разом з рештою:
`{"imp__strategy": ["mean", "median", "most_frequent"]}` — тоді порівняння
стратегій автоматично відбувається на одній схемі CV (`ml-search-strategy`).
