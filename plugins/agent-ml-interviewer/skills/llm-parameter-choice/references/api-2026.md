# API: параметри Claude, стан на 2026-07-18

**Джерело: актуальна довідка Anthropic API (claude-api skill середовища),
знято 2026-07-18.** Цей файл — єдине місце версійних фактів скіла; при оновленні
моделей правити тут. Половина порад з інтернету 2024–2025 років тут уже
**зламає запит** — звіряйте кожен параметр.

## Що ВИДАЛЕНО (повертає HTTP 400)

| Параметр | Де видалено | Заміна |
|---|---|---|
| `temperature`, `top_p`, `top_k` | Opus 4.8/4.7, Sonnet 5, Fable 5 | промпт-інструкції; structured outputs |
| `thinking: {type:"enabled", budget_tokens:N}` | ті самі | `thinking: {type:"adaptive"}` + effort |
| префіл останнього assistant-повідомлення | 4.6+ сім'я, Fable | `output_config.format` |
| `thinking: {type:"disabled"}` | лише Fable 5 | опустити параметр узагалі |

На **старіших** моделях (Haiku 4.5, Sonnet 4.5, Opus 4.5) семплінг і
budget_tokens досі чинні — тому «яка модель?» — перше питання скіла.

## Чинні контролі

```python
response = client.messages.create(
    model="claude-opus-4-8",
    max_tokens=16000,                       # >16k -- ТІЛЬКИ зі streaming
    thinking={"type": "adaptive",           # на 4.8 без поля = мислення вимкнене
              "display": "summarized"},     # дефолт "omitted" = порожній текст блоків
    output_config={"effort": "high"},       # low|medium|high|xhigh|max
    messages=[...],
)
```

- **effort** живе всередині `output_config`, не топ-рівнево. `xhigh` — дефолт
  Claude Code для кодингу/агентного.
- **Streaming**: `client.messages.stream(...)` + `get_final_message()`; для
  128k виходу — обов'язково.
- **Task budget** (beta `task-budgets-2026-03-13`, мін. 20000):
  `output_config={"task_budget": {"type":"tokens","total":64000}}` — модель
  бачить лічильник і завершує гідно, на відміну від жорсткого max_tokens.

## Кешування

```python
system=[{"type":"text","text": STABLE_PROMPT,
         "cache_control": {"type":"ephemeral"}}]        # 5 хв; {"ttl":"1h"} -- година
```

Економіка: запис ~1.25× (5м) / 2× (1г); читання ~0.1×; беззбитковість — 2 запити
(5м). Мінімальний префікс 1024–4096 токенів залежно від моделі. До 4 точок.
Перевірка: `response.usage.cache_read_input_tokens`. Порядок префікса:
tools → system → messages; будь-який змінений байт інвалідовує хвіст.

## Структуровані виходи

```python
client.messages.parse(..., output_format=PydanticModel)          # рекомендовано
# або messages.create(..., output_config={"format": {"type":"json_schema", "schema": {...}}})
```

Старий топ-рівневий `output_format` на `create()` — deprecated; канонічно —
`output_config.format`.

## Детермінованість і різноманіття без temperature

- Детермінованість: її не гарантував і `temperature=0`; для стабільної СТРУКТУРИ
  — structured outputs; для стабільного ЗМІСТУ — вужчий промпт + низький effort.
- Різноманіття: «Before building, propose 4 distinct directions… then implement
  only the chosen one» — робочий патерн замість temperature-лотереї.

## Оптимізація промптів

- **DSPy MIPROv2**: кандидати інструкцій + бутстреп few-shot демо → байєсівська
  оптимізація по метриці. `dspy.ai/api/optimizers/MIPROv2`.
- **GEPA** (Agrawal et al., 2026): рефлексія над трейсами виконання +
  еволюційний Парето-відбір; для компаундних систем.
- Передумова обох: заморожений eval-набір і метрика ДО оптимізації — інакше
  промпт переоснащується (аналог тюнінгу на test).

## Пастки

| Пастка | Наслідок | Захист |
|---|---|---|
| temperature/top_p на 4.8/Sonnet 5/Fable | 400 | прибрати |
| budget_tokens | 400 на 4.7+ | adaptive + effort |
| max_tokens 64k без stream | таймаут SDK | `.stream()` |
| datetime/UUID у системному промпті | кеш ніколи не читається | волатильне — після точки розриву |
| tools-список змінюється між запитами | інвалідація всього | стабільний, відсортований список |
| поради з блогів 2024–2025 | зламаний запит | звіряти з цим файлом/живою докою |
