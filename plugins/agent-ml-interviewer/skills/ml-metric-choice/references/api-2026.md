# API: метрики класифікації

**Перевірено 2026-07-18 проти встановленої scikit-learn 1.9.0** (оновлено з
1.6.1 того самого дня; усі факти нижче — живі виклики, не переказ документації).

Єдине місце у скілі з прив'язкою до версій.

## Імпорти

```python
from sklearn.metrics import (
    accuracy_score, balanced_accuracy_score,
    precision_score, recall_score, f1_score, fbeta_score,
    matthews_corrcoef, cohen_kappa_score,
    confusion_matrix, classification_report,
    roc_auc_score, average_precision_score,
    precision_recall_curve, roc_curve,
    make_scorer,
)
```

`average_precision_score` — це і є **PR-AUC**. Не рахуйте площу під PR-кривою
трапеціями вручну: `precision_recall_curve` дає ступінчасту криву, і трапеції
дають зміщену оцінку.

## Відповідність «рішення → виклик»

| Рішення зі SKILL.md | Виклик |
|---|---|
| accuracy | `accuracy_score(y, yhat)` |
| balanced accuracy | `balanced_accuracy_score(y, yhat)` |
| F-beta, β=2 | `fbeta_score(y, yhat, beta=2)` |
| F-beta, β=0.5 | `fbeta_score(y, yhat, beta=0.5)` |
| MCC | `matthews_corrcoef(y, yhat)` |
| Cohen's κ | `cohen_kappa_score(y1, y2)` |
| PR-AUC | `average_precision_score(y, y_score)` |
| ROC-AUC | `roc_auc_score(y, y_score)` |
| Gini | `2 * roc_auc_score(y, y_score) - 1` |
| усе по класах | `classification_report(y, yhat, digits=3)` |

## Важливо: мітки проти балів

| Метрика приймає | Аргумент |
|---|---|
| accuracy, BA, P, R, F, MCC, κ | **мітки** `y_pred` |
| ROC-AUC, PR-AUC | **бали** `y_score` (`predict_proba(X)[:, 1]`) |

Найпоширеніша помилка — передати `predict()` у `roc_auc_score`. Помилки не буде,
але AUC рахуватиметься по двох точках і виявиться заниженим.

## Усереднення в багатокласовій задачі

```python
f1_score(y, yhat, average=None)        # масив по класах -- дивіться першим
f1_score(y, yhat, average='macro')     # незважене середнє
f1_score(y, yhat, average='weighted')  # зважене за підтримкою
f1_score(y, yhat, average='micro')     # == accuracy у single-label задачі
```

`pos_label` для бінарної задачі, коли цікавий клас позначено `0`:

```python
f1_score(y, yhat, pos_label=0)
make_scorer(f1_score, pos_label=0)
```

## Використання в пошуку гіперпараметрів

```python
from sklearn.model_selection import GridSearchCV, cross_val_score

cross_val_score(est, X, y, cv=skf, scoring='roc_auc')
cross_val_score(est, X, y, cv=skf, scoring='average_precision')   # PR-AUC
cross_val_score(est, X, y, cv=skf, scoring='balanced_accuracy')
cross_val_score(est, X, y, cv=skf, scoring='matthews_corrcoef')

GridSearchCV(est, grid, scoring=make_scorer(fbeta_score, beta=2), cv=5)
```

Повний перелік рядкових імен: `sklearn.metrics.get_scorer_names()`.

## Зміни у 1.9 (усі підтверджено живими викликами)

| Що | Стан |
|---|---|
| `cohen_kappa_score(replace_undefined_by=...)` | **є з 1.9** — обробка виродженого 0/0 |
| `PrecisionRecallDisplay.from_cv_results` | **є з 1.9** — кілька PR-кривих із `cross_validate` |
| `metric_at_thresholds` | **є з 1.9** — метрики по всій сітці порогів |
| `d2_pinball_score`, `d2_absolute_error_score` | послідовно використовують `"averaged_inverted_cdf"` |

**Виродженість κ тепер обробляється параметром, дефолт уже безпечний:**

```python
cohen_kappa_score(y1, y2)                          # дефолт replace_undefined_by=nan
# UndefinedMetricWarning: "y1, y2 and labels have only one label in common.
# cohen_kappa_score is undefined and set to the value defined by ... nan"
cohen_kappa_score(y1, y2, replace_undefined_by=0.0)  # якщо потрібне конкретне число
```

Раніше (на 1.6.1) доводилось перевіряти `np.isnan(k)` вручну після виклику —
тепер це не потрібно, значення й так `nan`, а поведінку можна змінити одним
іменованим аргументом замість постобробки.

## Візуалізація

```python
from sklearn.metrics import (
    ConfusionMatrixDisplay, RocCurveDisplay, PrecisionRecallDisplay,
)

ConfusionMatrixDisplay.from_predictions(y, yhat, normalize=None)   # абсолютні числа
PrecisionRecallDisplay.from_estimator(est, X, y)                   # при дисбалансі
RocCurveDisplay.from_estimator(est, X, y)
```

Матрицю плутанини подавайте **в абсолютних числах** (`normalize=None`) — саме вони
показують масштаб задачі; нормовану додавайте другою, якщо треба.

## Що НЕ використовувати

| Хибне | Чому | Заміна |
|---|---|---|
| `accuracy_score` при частці класу < 10% | вироджена модель виграє | MCC або BA |
| `roc_auc_score(y, y_pred)` | бали замінено мітками | `predict_proba(X)[:,1]` |
| ROC-крива при величезному TN | FPR розбавлений | PR-крива |
| micro-F1 **і** accuracy як дві цифри | у single-label вони рівні | одна з них |
| `np.trapezoid` під PR-кривою | крива ступінчаста | `average_precision_score` |
