## Чому гармонійне середнє

F1 = гармонійне середнє precision і recall:

```
F1 = 2 / (1/P + 1/R) = 2PR / (P + R)
```

Мотивація з фізики (Лекція 2): середня швидкість на двох ділянках однакової
довжини — це `2/(1/v₁ + 1/v₂)`, а не `(v₁+v₂)/2`. Гармонійне середнє домінується
меншим доданком.

Наслідок — воно карає перекіс:

| P | R | арифметичне | гармонійне (F1) |
|---|---|---|---|
| 0.9 | 0.1 | 0.500 | **0.180** |
| 0.5 | 0.5 | 0.500 | 0.500 |
| 0.6 | 0.4 | 0.500 | 0.480 |

Класифікатор, що жертвує recall заради precision, не отримає високого F1.

## F-beta і правило β² < 1

```
Fβ = (1 + β²)·P·R / (β²·P + R)
```

Часткові похідні (Лекція 2 рахує їх символьно):

```
∂Fβ/∂P = (1+β²)·R² / (β²P + R)²
∂Fβ/∂R = (1+β²)·β²·P² / (β²P + R)²
```

Відношення чутливостей у симетричній точці P = R:

```
(∂Fβ/∂R) / (∂Fβ/∂P) = β²·P² / R² = β²   при P = R
```

Отже **β² — це у скільки разів F реагує сильніше на зміну recall, ніж precision**:

- `β² < 1` (β < 1) → precision домінує
- `β² = 1` (β = 1) → порівну, це F1
- `β² > 1` (β > 1) → recall домінує

Практичні значення: **β = 0.5** (precision вчетверо важливіша), **β = 2** (recall
вчетверо важливіший).

Сценарії з Лекції 2:

| Сценарій | Що дорожче | β |
|---|---|---|
| Псувний товар на складі | зайве замовлення псується → FP дорожчий | ½ |
| Контроль в аеропорту | пропустити загрозу → FN дорожчий | 2 |

В обох випадках зсув порогу простежується явно: FN↓ ⟹ FP↑ ⟹ Precision↓.

Історія назви: van Rijsbergen ввів E-функцію (1975); Chinchor застосувала на MUC-4
(1992); Sasaki, *The truth of the F-measure* (2007) зазначає, що літеру «F» було
обрано випадково.

## Провал accuracy при дисбалансі

```
Accuracy = (TP + TN) / (TP + TN + FP + FN)
```

Приклад Лекції 2. Реальний класифікатор:

```
TP = 20,  TN = 1000,  FP = 50,  FN = 5      (усього 1075)
Accuracy = (20 + 1000) / 1075 = 1020/1075 = 0.94884  →  94.88%
```

Вироджений класифікатор, який завжди відповідає «негативний»:

```
TP = 0,  FP = 0,  TN = 1000 + 50 = 1050,  FN = 20 + 5 = 25
Accuracy = 1050/1075 = 0.97674  →  97.67%
```

**Марна модель має вищу accuracy.** Це не парадокс, а визначення: accuracy
винагороджує відтворення апріорного розподілу.

Діагностика: порівняйте accuracy з часткою більшого класу. Якщо перевищення
менше кількох пунктів — метрику обрано неправильно.

## MCC

Matthews Correlation Coefficient — кореляція Пірсона між передбаченнями і
істиною, зведена до бінарного випадку:

```
MCC = (TP·TN − FP·FN) / √((TP+FP)(TP+FN)(TN+FP)(TN+FN))
```

Властивості, через які його варто брати при дисбалансі:

1. **Використовує всі чотири комірки**, на відміну від P, R, F1, які ігнорують TN.
2. **Симетричний щодо перейменування класів** — заміна «позитивний» ↔ «негативний»
   не змінює значення.
3. **Діапазон [−1, 1]**: 1 — ідеал, 0 — рівень випадкового, −1 — повна інверсія.
4. **Стійкий до вибору порогу.** У прикладі зі спам-фільтром Лекції 2 зміна порогу
   0.7 → 0.8 → 0.9 змінює accuracy, precision, recall і F1, але **не MCC**.

Багатокласове узагальнення — статистика **R_K** (Gorodkin, 2004). Увага:
мінімальне значення R_K вже не −1, а залежить від розподілу класів.

На аналітичній модельній задачі (f₀=2−2x, f₁=2x) MCC(t) = √(t(1−t)), максимум ½
при t = ½.

## Cohen's kappa

Міра згоди з поправкою на випадковий збіг:

```
κ = (p₀ − pₑ) / (1 − pₑ)
```

де `p₀` — спостережувана частка згоди, `pₑ` — очікувана за незалежності.

Приклад Лекції 2 (кредитний комітет, 50 заявок):

```
p₀ = 0.7,   pₑ = 0.5
κ = (0.7 − 0.5) / (1 − 0.5) = 0.2 / 0.5 = 0.4
```

Тобто 70% згоди виглядають добре, але половину з них пояснює випадок; реальна
згода понад випадкову — 0.4.

**Вироджений випадок:** якщо `pₑ = 1`, знаменник нульовий і κ невизначена.
scikit-learn 1.9 додав `cohen_kappa_score(replace_undefined_by=...)`.

Ланцюг джерел: Cohen (1960) → Rogot & Goldberg (1966) → Cohen (1968, зважена κ)
→ Conger (1980, багато оцінювачів) → Gwet (2014).

На модельній задачі κ(t) = 2t(1−t), максимум ½ при t = ½.

## Balanced accuracy

```
BA = (Sensitivity + Specificity) / 2 = (TPR + TNR) / 2
```

Ключова тотожність: **J = 2·BA − 1**, де J — індекс Юдена. Тому максимізація BA
тотожна критерію Юдена (див. `ml-decision-threshold/references/derivations.md`).

BA не залежить від часток класів, бо обидва доданки — умовні ймовірності. Саме це
робить її придатною при дисбалансі, на відміну від accuracy.

На модельній задачі BA(t) = (1 + 2t − 2t²)/2, максимум ¾ при t = ½.

## PR проти ROC при дисбалансі

Приклад Лекції 2 (спам, ~10⁶ листів, 100 справжніх спамів):

| | TP | FN | FP | TN | FPR | Precision |
|---|---|---|---|---|---|---|
| Алгоритм 1 | 90 | 10 | 10 | 999 890 | 0.00001 | **0.900** |
| Алгоритм 2 | 90 | 10 | 1 910 | 997 990 | 0.00191 | **0.045** |

Кількість хибних спрацювань зросла у 191 раз. ROC-крива ледь ворухнулась (FPR з
10⁻⁵ до 1.9·10⁻³ — обидва візуально нуль). Precision обвалилась удвадцятеро.

Причина: FPR має в знаменнику всі негативні (999 900), а precision — лише
передбачені позитивні (2 000). Коли TN величезний, він «розбавляє» FPR.

**Правило: якщо TN величезний і нецікавий — беріть PR-криву.**

Джерело: Saito T., Rehmsmeier M. (2015), *PLoS One*.
