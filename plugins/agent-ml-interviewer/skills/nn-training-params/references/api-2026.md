# API: навчання нейромереж

**Перевірено 2026-07-18 живими викликами: keras 3.12.0 (backend — tensorflow
2.20.0), scikit-learn 1.9.0.** Сигнатури зчитані `inspect.signature`, міні-прогін
навчання виконано реально (6 епох, EarlyStopping, history з `val_loss`/`val_AUC`).

**`torch` у середовищі ВІДСУТНІЙ** — приклади на PyTorch тут не перевіряються й
не наводяться. Доступні: `keras` (3.12.0, backend tensorflow) і
`sklearn.neural_network.MLPClassifier`.

## Мінімальний робочий каркас (перевірено прогоном)

```python
import keras

model = keras.Sequential([
    keras.layers.Input((n_features,)),
    keras.layers.Dense(64, activation="relu"),
    keras.layers.Dropout(0.2),
    keras.layers.Dense(1, activation="sigmoid"),
])
model.compile(optimizer=keras.optimizers.Adam(learning_rate=1e-3),
              loss="binary_crossentropy", metrics=["AUC"])

hist = model.fit(X, y, validation_split=0.25, epochs=200, batch_size=32, verbose=0,
                 callbacks=[keras.callbacks.EarlyStopping(
                     monitor="val_loss", patience=10, restore_best_weights=True)])
```

`hist.history` містить ключі `loss`, `val_loss` і по одному на метрику
(`AUC`, `val_AUC`) — це і є криві навчання для `ml-overfitting-diagnosis`.

## Точні сигнатури (keras 3.12.0)

| Об'єкт | Параметри |
|---|---|
| `EarlyStopping` | `monitor, min_delta, patience, verbose, mode, baseline, restore_best_weights, start_from_epoch` |
| `ReduceLROnPlateau` | `monitor, factor, patience, verbose, mode, min_delta, cooldown, min_lr` |
| `Adam` | `learning_rate, beta_1, beta_2, epsilon, amsgrad, weight_decay, clipnorm, clipvalue, global_clipnorm` |

- **`restore_best_weights=True` — не дефолт.** Без нього модель лишається у стані
  ОСТАННЬОЇ епохи, а не найкращої: раннє спинення відпрацює, а ваги будуть гірші.
- `start_from_epoch` — не рахувати patience перші N епох (корисно, коли на
  початку метрика шумна).
- L2 у сучасному Keras задається як **`weight_decay` в оптимізаторі**
  (`Adam(weight_decay=1e-4)`) або `kernel_regularizer` на шарі.

## Розклад learning rate

```python
keras.callbacks.ReduceLROnPlateau(monitor="val_loss", factor=0.5,
                                  patience=5, min_lr=1e-6)
```

Це і є правило «тримати η сталим, доки валідація не почне гіршати, тоді ділити»
(`factor=0.5` — ділення на 2, `factor=0.1` — на 10).

## Пошук порога learning rate

```python
for lr in [1e-4, 1e-3, 1e-2, 1e-1, 1.0]:
    m = build_model()
    m.compile(optimizer=keras.optimizers.Adam(learning_rate=lr), loss="binary_crossentropy")
    h = m.fit(X, y, epochs=5, batch_size=32, verbose=0)      # ДИВИТИСЬ train loss
    print(lr, [round(v, 4) for v in h.history["loss"]])
```

Шукається найбільший `lr`, при якому train-вартість ще **спадає** в перші епохи;
робоче значення — удвічі менше. Валідація тут не потрібна (див. SKILL.md, Крок 2).

## Порівняння розмірів батча за часом

```python
import time
for bs in [16, 32, 64, 128, 256]:
    t0 = time.perf_counter()
    h = build_model_compiled(lr=base_lr * bs / 32).fit(  # η масштабується з батчем
```

Порівнювати треба метрику проти **секунд**, не проти епох.

## sklearn MLPClassifier (коли Keras надлишковий)

Перевірено наявність параметрів: `hidden_layer_sizes`, `learning_rate_init`,
`early_stopping`, `n_iter_no_change`, `validation_fraction`, `alpha`, `batch_size`.

```python
from sklearn.neural_network import MLPClassifier
MLPClassifier(hidden_layer_sizes=(64, 32), learning_rate_init=1e-3,
              early_stopping=True, n_iter_no_change=10, validation_fraction=0.15,
              alpha=1e-4, batch_size=32, max_iter=500, random_state=0)
```

Відповідність понять: `alpha` = L2-λ; `n_iter_no_change` = patience;
`early_stopping=True` вмикає внутрішній валідаційний спліт. Перевага — працює в
`Pipeline` і `GridSearchCV` як звичайний естиматор (`ml-search-strategy`), тож
для табличних задач це найдешевший спосіб спробувати мережу.

## Скейлінг обов'язковий

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
Pipeline([("sc", StandardScaler()), ("nn", MLPClassifier(...))])
```

Для Keras — той самий scaler, зафічений **тільки на train** (інакше витік; див.
`ml-validation-design`).
