# API: модель вимірювання й надійність — перевірено 2026-07-19

Середовище: **Python 3.12.10 · scikit-learn 1.9.0 · numpy 2.2.6 ·
scipy 1.16.3 · statsmodels 0.14.6 · pandas**. Усе нижче — живі виклики.

## 0. Чого в середовищі НЕМАЄ (перевірено імпортом)

| Пакет | Що дав би | Заміна в паку |
|---|---|---|
| `factor_analyzer` | поліхорична, обертання, KMO/Bartlett | власна `polychoric()` у скрипті |
| `semopy` | багатогрупова CFA, формальний тест інваріантності | `invariance_probe()` — детектор, не тест |
| `pingouin` | `cronbach_alpha` з ДІ | `cronbach_alpha()` у скрипті |

**Це не дрібниця.** Формального тесту інваріантності в середовищі немає, і
казати користувачеві «перевірте інваріантність» без вказівки, чим саме, —
порожня порада. Скрипт дає детектор; повний тест вимагає встановлення `semopy`
(або R/`lavaan`), і про це треба сказати прямо.

## 1. `FactorAnalysis` — що воно рахує і чого не рахує

```python
from sklearn.decomposition import FactorAnalysis
fa = FactorAnalysis(n_components=1, random_state=0).fit(Xz)
fa.components_        # (n_components, n_features) — навантаження
fa.noise_variance_    # (n_features,) — унікальні дисперсії psi
fa.transform(Xz)      # факторні бали (регресійний метод)
```

- **Ознаки треба стандартизувати вручну, і ціна помилки більша, ніж здається.**
  У коді є `X -= self.mean_`, але жодного ділення на `std` — перевірено читанням
  `fit` і живим викликом. Наслідок не косметичний: колонка з найбільшою
  дисперсією **перетягує фактор на себе**, як у PCA на нескейлених даних.

  Три ознаки з **ідентичною** істинною структурою (комунальність 0.64) у
  масштабах ×1, ×10, ×100:

  | | ознака ×1 | ознака ×10 | ознака ×100 |
  |---|---|---|---|
  | без стандартизації | 0.4061 | 0.4030 | **0.9999** |
  | зі стандартизацією | 0.6436 | 0.6389 | 0.6309 |

  Найбільша колонка забрала практично всю спільну дисперсію, дві інші занижені
  з 0.64 до 0.40. Тотожність `λ² + ψ = Var(X)` при цьому виконується точно —
  тобто **діагностика «сума збіглася» помилки не покаже**.
- **Знак фактора довільний.** `components_` може вийти цілком від'ємним; перед
  використанням нормалізуйте знак (наприклад, за `lam.sum() < 0`).
- **`transform` дає регресійні бали (Thomson), не Бартлета.** Різниця не
  косметична: регресійні зміщені до середнього (shrunk), Бартлета — незміщені,
  але шумніші. sklearn не пропонує вибору.
- **`noise_variance_` не буває від'ємною** — `np.maximum(..., SMALL)` з
  `SMALL = 1e-12`. Перевірено: підлога не досягається навіть при агресивному
  над-витягу (0/1200, деталі в `derivations.md`). **Не покладайтесь на це як
  на детектор недопустимого розвʼязку.**
- `rotation="varimax"` / `"quartimax"` доступні; за замовчуванням `None`.

## 2. Кореляційні матриці та їх допустимість

```python
import numpy as np
np.linalg.eigvalsh(corr).min() > 0     # ознака позитивної визначеності
np.linalg.cholesky(corr)               # кине LinAlgError, якщо ні
```

**`pandas.DataFrame.corr()` на даних з NaN використовує попарне видалення** —
кожна пара рахується на своїй підвибірці. Це найпоширеніша причина
непозитивно визначеної матриці (виміряно 10% на MAR-пропусках).

```python
df.corr()                    # попарне видалення (дефолт) — небезпечно
df.dropna().corr()           # спискове — завжди допустима, але втрата n
df.corr(min_periods=50)      # НЕ рятує: лишає NaN, не робить матрицю PD
```

`min_periods` лише замінює ненадійні кореляції на `NaN` — матриця стає ще
менш придатною, а не більш. Правильний шлях — імпутація (`ml-missing-data`)
або спискове видалення.

## 3. `GaussianMixture` і `reg_covar`

```python
from sklearn.mixture import GaussianMixture
gm = GaussianMixture(n_components=k, reg_covar=1e-6).fit(X)   # дефолт 1e-6
np.linalg.det(gm.covariances_[j])          # порівняти з reg_covar**d
gm.weights_.min() * len(X)                 # скільки точок у найменшій компоненті
```

- `reg_covar` **не логує й не попереджає**. Виродження діагностується
  вручну: `det(Σ) ≈ reg_covar**d` і вага компоненти ≈ 1-2 спостереження.
- `gm.converged_ == True` при виродженні теж `True` — воно **не є** ознакою
  допустимості.
- Вироджений розвʼязок має **вищу** правдоподібність, тому `bic()`/`aic()` на
  великих `k` можуть указувати саме на нього. Перевіряйте `min det` окремо.

## 4. Дискретизація: що саме робить sklearn

```python
from sklearn.preprocessing import KBinsDiscretizer
KBinsDiscretizer(n_bins=5, encode="ordinal", strategy="quantile")
```

- `strategy="quantile"` — рівноймовірні категорії; це **найсприятливіший**
  випадок за втратою кореляції (числа в `derivations.md` виміряні саме на
  ньому). `strategy="uniform"` на скошених даних втрачає більше.
- `subsample` за замовчуванням `200_000` від версії 1.3 — на великих даних
  пороги рахуються на підвибірці; для відтворюваності задавайте `random_state`.
- **`encode="ordinal"` створює порядкову змінну, яку більшість моделей
  прочитає як числову з рівними інтервалами** — ще одне непомічене припущення
  вимірювання поверх огрублення.

## 5. Поліхорична кореляція власними силами

Реалізація в `measurement_model_report.py` (`polychoric()`):

```python
from scipy import optimize, stats
stats.multivariate_normal(mean=[0,0], cov=[[1,rho],[rho,1]]).cdf([a, b])
optimize.minimize_scalar(negll, bounds=(-0.99, 0.99), method="bounded")
```

- `multivariate_normal.cdf` у scipy 1.16 приймає точку як список — саме через
  неї рахується ймовірність комірки за формулою включення-виключення.
- Оптимізація **одновимірна** (лише `rho`, пороги фіксовані з маргіналів), тому
  стійка й не потребує стартових значень.
- **Порожні комірки** таблиці спряженості пропускаються (`if tab[i,j] == 0`) —
  інакше `log(0)`. На рідкісних категоріях це знижує точність; при повністю
  порожньому рядку/стовпці категорію треба злити з сусідньою.
- Швидкодія: `k=5` → 25 комірок × ~30 ітерацій. Для матриці 20×20 ознак це
  190 пар — секунди, не мілісекунди. Для великих матриць кешуйте.

## 6. Alpha і omega вручну

```python
k = X.shape[1]
alpha = k/(k-1) * (1 - X.var(axis=0, ddof=1).sum() / X.sum(axis=1).var(ddof=1))
```

**Спростована власна гіпотеза про `ddof`.** Я записав був, що `ddof=0` зсуває
alpha і що на малій вибірці це помітно. **Перевірка показала протилежне:** на
n=50 різниця `1.11e-16`, тобто машинний нуль. Причина очевидна заднім числом —
alpha є **відношенням** дисперсій, і множник `(n−1)/n` скорочується в чисельнику
й знаменнику точно. `ddof` тут не має значення; вимога узгодженості лишається
стилістичною, не числовою.

Omega рахується з навантажень (`omega_total(lam, psi)`), тобто вимагає
факторної моделі. `omega_from_data()` у скрипті робить обидва кроки.

## 7. Порядок операцій у Pipeline

Композит — це **трансформація ознак**, отже їй місце всередині Pipeline під CV,
як селектору чи скейлеру (`ml-validation-design`):

```python
from sklearn.preprocessing import FunctionTransformer
Pipeline([("composite", FunctionTransformer(lambda X: X.mean(1, keepdims=True))),
          ("clf", LogisticRegression())])
```

Якщо ваги композита оцінюються з даних (факторний бал, оптимальні λ), то
оцінювати їх **на всій вибірці до спліту — витік**, тієї самої природи, що й
скейлер на всіх даних. Незважене середнє витоку не дає (ваги задані апріорі) —
це єдина його реальна перевага перед зваженим.
