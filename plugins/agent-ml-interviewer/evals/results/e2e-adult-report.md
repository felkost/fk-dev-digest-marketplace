# Наскрізний прогін оркестратора ml-tuning-workflow — Adult — 2026-07-18

Повний цикл за контрактом скіла: анкета → метрика → спліт → базлайн → діагноз →
канарки/VIF → сімейні ручки → пошук → поріг → **один** дотик запечатаного test.
Кожен крок виконано інструментами пака; артефакти — scratchpad/eval/e2e_*.

## Анкета підпису (крок 0)

| Питання | Відповідь |
|---|---|
| Задача | бінарна класифікація (дохід >50k) |
| Дані | `adult_data_witn_att_names01.csv`: **32 561 × 15**; 6 числових, 8 категорійних |
| Пастки читання | `sep=';'` ✓; провідні пробіли (`' State-gov'`→`'State-gov'`) ✓; ціль уже 0/1 ✓ |
| «?» | workclass 1836, occupation 1843, native-country 583 → **окрема категорія**, без мовчазної імпутації |
| Баланс | позитивних **24.1%** → помірний дисбаланс (10–40%) |
| Групи/час | немає → StratifiedKFold |
| Ціна помилок | не задана бізнесом → пороги показуємо віялом критеріїв |

## Метрика (крок 1, за таблицею ml-metric-choice)

Помірний дисбаланс без заданої асиметрії → **основна PR-AUC** (`average_precision`)
для пошуку, **MCC/BA** і матриця плутанини — у звіт. Accuracy не звітується.

## Спліт (крок 2)

`train_test_split(test_size=0.2, stratify=y, seed=42)` → train 26 048 /
**test 6 513 запечатано**; всередині — `StratifiedKFold(5, shuffle, seed=42)`.

## Діагноз (крок 4, скрипти пака)

**4а. Свідомо перенавчене дерево** (без обмежень) → `overfit_report.py`:
MCC train 0.9999 / val 0.4877, розрив **+51.2%** → «СИЛЬНЕ ПЕРЕНАВЧАННЯ (>30%)»,
дія «різати ємність» — вердикт і маршрут скрипта збіглися з очікуваними.

**4б. Канарки** (`noise_sanity_check.py`, 12 шумових, RF): CV ROC-AUC 0.8271 →
0.8535 після сміття, **Δ = +0.0264 > SE 0.0048 → прапор переоснащення
протоколу**. `fnlwgt` на 5-му місці (0.0443) — **нерозрізненний із найкращою
канаркою (0.0439)**: урок нотбука курсу відтворено на живих даних (метадана
перепису без причинного механізму). `capital-loss` — нижче канарки. Наслідок для
пайплайна: важливостям RF тут не вірити без permutation; на числових ознаках
відбору за importance не робити.

**4в. VIF** (`collinearity_report.py`, з константою): усі 6 числових **1.01–1.04**
— колінеарності немає, прунити нічого. Чесний негативний результат.

## Сімейні ручки + пошук (кроки 6–7)

Модель за `ml-tree-ensemble-params`: **HistGradientBoostingClassifier** із
нативними категоріями (`categorical_features="from_dtype"`, без one-hot),
рання зупинка. Пошук за `ml-search-strategy`: RandomizedSearchCV, **12
кандидатів × 5 фолдів = 60 фітів (62 с)**, lr на лог-шкалі.

- **Переможець:** `learning_rate=0.0535, max_leaf_nodes=31, l2=0.0,
  min_samples_leaf=20` → CV PR-AUC **0.8254** (train 0.8778, розрив 6.0% — норма).
- **Правило 1 SE:** у коридорі 2 конфігурації; найпростіша —
  `lr=0.094, leaves=31, l2=0.1` (0.8240) — рекомендована до прод-розгляду.
- Проти базлайна: +0.063 PR-AUC над логістичною.

## Запечатаний test (крок 9 — один дотик)

**PR-AUC 0.8356, ROC-AUC 0.9306** (CV обіцяла 0.8254 — узгоджено, зсуву немає).

| Поріг | Precision | Recall | MCC | Матриця (TP/FP/FN/TN) | на 1000: FP / FN |
|---|---|---|---|---|---|
| youden 0.208 | 0.591 | **0.894** | 0.620 | 1401/970/167/3975 | 148.9 / 25.6 |
| fbeta 0.402 | 0.723 | 0.753 | **0.653** | 1181/452/387/4493 | 69.4 / 59.4 |

## Межі

- Ціна FP/FN не задана бізнесом — фінальний поріг лишається рішенням замовника;
  віяло вище — це меню з цінниками (на 1000 осіб).
- Важливості RF на цих даних недостовірні (канарковий прапор) — інтерпретацію
  будувати через permutation importance, не через `feature_importances_`.
- «?» лишено категорією; альтернативні стратегії не порівнювались.

## Позначки тверджень

Факти: всі числа вище (пораховано в сесії). Інтерпретація: причина канаркового
прапора (переоснащення протоколу RF на додаткових вимірах). Припущення: пороги
курсу 15/20/30% застосовні до MCC так само, як до R²/RMSE.
