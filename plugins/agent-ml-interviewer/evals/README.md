# Перевірка пака

## Рівень A — аналітична основна істина (без LLM, детермінована)

Найцінніша перевірка в паку: Лекція 2 виводить **замкнені форми**, тож
правильність скрипта перевіряється точно, а не «на око».

```
python ml-decision-threshold/scripts/threshold_report.py --self-test
```

Очікується (модель f₀ = 2−2x, f₁ = 2x на [0,1], **рівні апріорні ймовірності**):

| Величина | Аналітично | Джерело |
|---|---|---|
| аргмакс F1 по порогу | t\* = (3−√5)/2 ≈ 0.381966 | корінь t² − 3t + 1 = 0 |
| максимум F1 | 3 − √5 ≈ 0.763932 | підстановка t\* |
| AUC-ROC | 5/6 ≈ 0.833333 | ∫₀¹ 2y(2y − y²) dy |
| AUC-PRC | 5/6 | ∫₀¹ ½(1 + √(1−R)) dR |
| поріг за Youden | ½ | максимум BA(t) = (1+2t−2t²)/2 |
| BA у цій точці | ¾ | там само |

Останній прогін (2026-07-18, n = 400 000): F1 = 0.763903 (похибка −3·10⁻⁵),
AUC = 0.833198, Youden t = 0.4892, BA = 0.7499. Тотожність J = 2·BA − 1
виконується на всій сітці порогів.

**Що ця перевірка ловить:** помилки у виводах — найшкідливіший і найменш помітний
клас помилок у паку порад.

## Рівень A2 — звірка зі sklearn (незалежна реалізація)

Фікстура `golden/scores_imbalanced.csv` — 2400 балів LogisticRegression на
незбалансованих даних (7.3% позитивних). Перевірено 2026-07-18:

- мій `fbeta` проти `sklearn.metrics.fbeta_score`: **0 розбіжностей** на 40 порогах
  для β ∈ {0.5, 1, 2};
- мій критерій `youden` проти аргмаксу `sklearn.metrics.balanced_accuracy_score`:
  **обидва дають t = 0.1545** — тотожність J = 2·BA − 1 підтверджена на даних;
- монотонність за β: 0.5 → 1 → 2 → 4 дає пороги 0.343 → 0.290 → 0.155 → 0.063,
  recall 0.64 → 0.66 → 0.74 → 0.85 (більша β тягне поріг униз заради recall).

### Попередній прогін (27 пасток, v8)

**b24-b26 на зонах восьмої звірки: базлайн 3/3, зі скілом 3/3, дельта +0;

**Два з трьох прогнозів, записаних ДО прогону, спростовано** (b24 і b26
очікувались як +1). Базлайн відповів правильно на всі три й не поверхнево — сам
назвав приховане припущення каліброваності, самостійно вивів, що GroupKFold може

межах правильно специфікованої родини. **Шоста й сьома поспіль спростовані
«+1»-гіпотези цього класу:** дельту дає лише ЗАСТАРІЛІСТЬ знань моделі щодо
середовища пака, а не контрінтуїтивність факту.

**Чотири дефекти лінійки, знайдені прогоном — усі в маркерах, не в паку:**

1. **Хибний PASS** на b25, спійманий adversarial-файлом ДО живого прогону:
   `лише\s+з?\s*train` збігався з «бачить лише train-фолди» — твердженням про
   фолди, а не про мітки.
2. **Хибний FAIL** на b25: `правильно\s+діагносту` без `\b` збігався всередині
   «**не**правильно діагностує». Шостий випадок цього класу в історії пака.
3. **Хибний FAIL на b26 в ОБОХ гілках:** маркери `чесно поверне` / `BIC захист`
   узяті дослівно з умови, тож спрацьовували при цитуванні колеги задля
   спростування. **Правило на майбутнє: wrong-маркер не має повторювати
   формулювання з промпту — він має ловити сам акт згоди моделі.**
4. **Найважливіший — маркери були прив'язані до ЛЕКСИКИ СКІЛА.** Після правок
   (1)-(3) матриця показала дельту **+2**, і це число було фальшивим: базлайн
   відповів правильно, але іншими словами («out-of-fold», «fold-aware»,
   «апроксимує» замість «трансдуктивна», «апроксимат»). Лінійка міряла б «чи
   прочитав ти мій документ». `correct_any` переписано на рівень ПОНЯТТЯ.
   Небезпека цього класу в тому, що помилка працює **на користь** пака, тож її
   не хочеться шукати.

значущості» — p-hacking'ом. Цінність цих двох пасток регресійна: вони фіксують
виправлену помилку на майбутнє.

Урок про валідацію маркерів: b9 спершу дав хибний FAIL, бо маркери
валідувались на **синтетичній відповіді, яку писав той самий автор** — і
збіглись із його формулюваннями, а не з простором можливих. Синтетична
фікстура перевіряє, що маркер узагалі стріляє; що він стріляє по чужому
тексту — перевіряє лише живий прогін.

### Виправлення грейдера 2026-07-18 (впливає на порівнянність)

Знайдено при валідації b10: у списку заперечної лексики бракувало **меж слова**,
тож `не\s`/`ні\s` збігались із закінченнями прикметників — «унікал**ьні** »,
«сил**ьне** ». Такі слова є майже в кожному вікні ±200 символів українського
тексту, тобто wrong-маркери глушились **майже завжди**, і хибна відповідь могла
пройти. Додано `\b`; заодно прибрано зі списку `прибер` — в API-пастках це
коректне застереження («приберіть `gamma=`»), а в b10 «приберіть предиктор» —
сама хибна порада, тож маркер глушив сам себе.

Наслідок: **записаний результат 6/8 отримано за старим, м'якшим правилом** і
один-в-один не відтворюється (файли відповідей не збережені — вони були в
scratchpad поза деревом). Числа старої й нової матриць не змішувати; при
наступному живому прогоні переганяти всі 10 пасток заново. Перевірено, що
виправлення не ламає старі пастки: синтетичні коректні відповіді з «Явно НЕ
додавайте gamma=…», «base_estimator видалено», «temperature… поверне 400»

## Методологія рівня C (збережено як довідку)

**Головна пастка методології.** Базлайн «без скіла», якому поставили ML-питання,
відповість правдоподібно — модель і так знає, що таке F1. Якщо тестувати загальну
компетентність, дельта «зі скілом / без» схлопнеться в нуль, і harness покаже,
що пак нікчемний. Це буде **похибка вимірювання, а не висновок**.

Тестувати треба там, де апріорні знання моделі **застарілі**:

| Промпт-пастка | Правильна відповідь | Що модель скаже без скіла |
|---|---|---|
| «яку temperature для Opus 4.8» | параметра немає, поверне 400 | «спробуйте 0.7» |
| «підбери поріг» | `TunedThresholdClassifierCV` | ручний перебір по roc_curve |
| «налаштуй Optuna» | без `gamma`/`weights` (deprecated 4.9) | `TPESampler(gamma=..., weights=...)` |
| «gradient boosting» | без `criterion` (deprecated 1.9) | `criterion='friedman_mse'` |
| «порахуй VIF» | з константним стовпцем | без нього → тихо неправильні числа |
| «SVC із ймовірностями» | `CalibratedClassifierCV(ensemble=False)` | `SVC(probability=True)` |
| «порівняй моделі при 1% позитивних» | PR-AUC | ROC-AUC |

Другий клас (b9-b10) працює інакше: хибне твердження **вкладено в промпт**, і
міряється не знання, а готовність його виправити. Тут базлайн може пройти —
і це нормально.

| Промпт-пастка | Правильна відповідь | Спосіб провалитись |
|---|---|---|
| «Шапіро-Уїлк відхилив нормальність залишків — OLS більше не BLUE?» | нормальність не входить до умов Гаусса-Маркова; BLUE стоїть, страждає лише точний t/F-інференс | підтвердити «п'яту умову» і відправити лікувати залишки |
| «жоден предиктор не значущий при R²=0.42, VIF 3-4 — що прибрати?» | строгої колінеарності немає; коефіцієнти незміщені; це результат, а не збій | назвати винною мультиколінеарність і порадити викинути предиктор заради значущості |

**Друга пастка:** фікстури мають лежати **поза** деревом пака. Субагент із
робочою текою всередині проєкту знайде `.claude/skills/` і застосує скіл сам —
базлайн буде контамінований, а в транскрипті це невидно.
