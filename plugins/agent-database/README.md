# agent-database: викладач-аналітик баз даних

Плагін-агент зі скілами для **курсу баз даних**: аналізує умови завдань, рецензує готові
SQL/CQL-запити в шести діалектах, вчить будувати запити в SQLite і MongoDB, пояснює
під'єднання з Python / Docker / хмари, проєктування DWH за стандартами, ETL і lakehouse,
BI-аналітику. Аудиторія — **викладач**: повні еталонні розв'язки, критерії оцінювання й розбір
типових помилок студентів (не режим «підказок без відповіді»).

```
sql-mentor (агент)            ← персона викладача, маршрутизація, контракт виводу, кураторський режим
├── analyze-task-conditions      ← розбір умови до написання SQL (+ нормалізація, BABOK)
├── analyze-sql-examples         ← рецензія готового запиту: 6 діалектів + каталог граблів + банк задач
├── explain-sqlite-mongodb       ← SQLite і MongoDB з нуля, SQL ↔ aggregation pipeline
├── db-connectivity-cloud        ← Python, Docker, AWS/GCP/Azure, GraphQL, Redis, векторні БД і RAG
├── build-data-projects          ← ETL/аналітичні конвеєри, lakehouse (Delta Lake, Databricks, Glue)
├── design-dwh-etl               ← схеми БД/DWH за стандартами (зірка, DMBOK), ETL/ELT, якість даних
└── bi-analytics                 ← Power BI (Power Query, модель, DAX) і Tableau (LOD, дашборди)
```

Кожен скіл: `SKILL.md` (методологія) + `references/*.md` (поглиблені довідники) — разом 43
файли знань. Плагін також містить Claude Code-агента
[`agents/sql-mentor.md`](agents/sql-mentor.md), що маршрутизує роботу через ці скіли.

**Підтримувані системи:** MySQL, Microsoft SQL Server (T-SQL), PostgreSQL, PL/pgSQL,
Cassandra (CQL), ClickHouse — аналіз; SQLite, MongoDB — навчання й побудова запитів.

Скіли відповідають **українською** (якщо не попросити іншу мову); SQL/CQL/Mongo-код лишається
мовою оригіналу.

## Контракт виводу

Кожна відповідь із розв'язком іде за фіксованою структурою — це головна дисципліна плагіна:

> **Середовище/діалект → Аналіз → Розв'язок (запит) → Пояснення → Ризики → Альтернативи →
> Обмеження**

За доречності додається блок **«Критерії оцінювання / типові помилки студентів»**. Просте
питання-факт відповідається стисло, але діалект позначається завжди.

## Кураторський режим (як росте база знань)

Агент **не** змінює свої файли сам — навмисно: так у базу не потрапляють неперевірені висновки.
Наприкінці аналізу, якщо є що узагальнити, він **пропонує**:

- **📌 Урок-кандидат** — узагальнене правило + конкретний файл, куди його додати, + статус
  (нове / уточнює наявне / вже покрито);
- **📒 Запис у банк задач** — повний розбір показової задачі за шаблоном
  [`analyze-sql-examples/references/task-bank.md`](skills/analyze-sql-examples/references/task-bank.md).

Рішення про внесення приймає людина. Так база росте контрольовано, без дрейфу.

## Стан перевірки

Усі 7 скілів пройшли перевірку реальними тестовими запитами через агента (не лише написані «на
віру»): у кожному прогоні агент підтягнув потрібні довідники, дав відповідь за контрактом
виводу, а знайдені прогалини — довантажено в базу (схвалені поповнення) або виправлено як
фактичні помилки. Банк показових задач налічує **8 розібраних кейсів**, серед них — регресійні
приклади саме з цих тестів (MongoDB `$group`, DWH для онлайн-школи, Power BI «три симптоми»,
pgvector-пошук, medallion на Delta Lake, as-of join на SCD2-вимір). Деталі й дати перевірок — у
[HANDOFF.md](HANDOFF.md).

## Межі (важливо)

- **DDL і приклади проєктування схем/ETL** подаються сумісними з **PostgreSQL, MySQL і
  ClickHouse**; специфіка ClickHouse (MergeTree/ORDER BY, відсутність FK) коментується завжди.
- **Перевіряй, коли можеш:** приклади SQLite прогоняються через `sqlite3` для підтвердження
  коректності. Для решти СУБД без реального сервера приклади ілюстративні — синтаксис
  звіряється з офіційною документацією, не вигадується.
- **Невиразність називається прямо:** якщо задача не виражається в цільовій системі (довільний
  JOIN чи віконна функція в Cassandra CQL) — це йде в «Обмеження» разом з обхідним шляхом.
- **Секрети — не в коді:** приклади під'єднання читають їх з env або секрет-менеджера.
- Строгість підготовки датасетів для ML (split / leakage / EDA) — парафія сусіднього плагіна
  `eda-skills`; цей плагін на нього посилається, а не дублює.
